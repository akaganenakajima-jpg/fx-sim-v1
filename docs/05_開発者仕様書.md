## 3. モジュール仕様

### 3.1 index.ts — エントリーポイント

**cron フロー（Ph.6 以降）**:
1. ワンタイムマイグレーション確認
2. `Promise.allSettled` で news / indicators / frankfurter を並列取得
3. ニュースソース統計を `news_fetch_log` にバッチ INSERT
4. 全銘柄の現在レートをマッピング（Yahoo 失敗時はキャッシュフォールバック）
5. 全ポジションの TP/SL チェック + トレイリングストップ
6. **Logic エントリー**: RSI + ER + ATR + 拡張パラメーター（vix_tp_scale 等）で自律判定
7. スキップ銘柄をバッチ INSERT（HOLD 記録）
8. BUY/SELL ならブローカー経由でポジション開設
9. **Path B**（ニュースハッシュ変化時のみ）: AI ニュース分析 → EMERGENCY/TREND 処理
10. **Param Review**（N 件取引ごと）: AI がパラメーターを最適化提案
11. 実行時間記録、30s 超過で WARN、ログパージ、日次サマリー

**HTTP ハンドラ**: 14 ルート（静的5 + アイコン4 + API5）

### 3.2 gemini.ts — AI API

**関数一覧（Ph.6 以降）**:

| 関数 | 用途 | タイムアウト | 状態 |
|------|------|-------------|------|
| ~~`getDecision()`~~ | ~~Gemini 銘柄判定~~ | — | **Ph.6 廃止** |
| ~~`getDecisionWithHedge()`~~ | ~~マルチモデル判定~~ | — | **Ph.6 廃止** |
| `analyzeNews()` | Gemini ニュース分析 | 8s | Path B で使用 |
| `analyzeNewsGPT()` | GPT ニュース分析 | 8s | フォールバック |
| `analyzeNewsClaude()` | Claude ニュース分析 | 8s | フォールバック |
| `callGeminiForReview()` | パラメーターレビュー | 15s | Param Review で使用 |

**プロンプト設計**: ニュース分析は構造化テキストで入力し JSON レスポンスを強制。Param Review は取引実績データ + 現行パラメーターを渡し最適化提案を取得。

### 3.2b logic-trading.ts — Logic エントリー（Ph.3〜6）

テクニカル指標のみでエントリー判定を行う（AI 不使用）。

- `evaluateEntry()`: RSI + ER + ATR で BUY/SELL/HOLD を判定
- TP/SL: `ATR × atr_tp_multiplier × vix_tp_scale`（VIX 条件分岐）
- SL: `× vix_sl_scale × macro_sl_scale`（マクロ下落時）
- `min_signal_strength` フィルタ: RSI偏差 + ER の平均が閾値未満ならスキップ
- `strategy_primary`: `trend_follow` 時は ER ≥ adx_min/100 を追加チェック

### 3.2c param-review.ts — AI パラメーターレビュー（Ph.4〜6）

N 件取引到達時に AI が全パラメーター（基本 10 + 拡張 5）を最適化提案。

- `runParamReview()`: 銘柄ごとにレビュートリガー判定 → AI 呼出 → 検証 → 反映
- `validateAndClamp()`: AI 提案値を ±20% 範囲 + 絶対上下限でクランプ
- `strategy_primary`: ホワイトリスト検証（'mean_reversion' | 'trend_follow'）

### 3.3 position.ts — ポジション管理

- `checkAndCloseAllPositions()`: 全 OPEN ポジションの TP/SL を一括チェック
- `openPosition()`: 新規ポジション作成（D1 INSERT）
- **トレイリングストップ**: `trailingActivation` 以上の含み益で SL を `trailingDistance` 幅に引上げ

### 3.4 broker.ts — ブローカー抽象化

**BrokerAdapter インターフェース**:
```typescript
interface BrokerAdapter {
  openPosition(params: OrderParams): Promise<BrokerResult>;
  closePosition(params: CloseParams): Promise<BrokerResult>;
  updateStopLoss(params: UpdateSLParams): Promise<BrokerResult>;
}
```

- **PaperBroker**: 何もしない（D1 記録は呼出元が行う）
- **OandaBroker**: OANDA REST API v20 で MARKET 注文（FOK）
- **getBroker()**: `TRADING_ENABLED` + 銘柄設定で自動ルーティング
- **withFallback()**: OANDA 失敗時にペーパーにフォールバック

### 3.5 risk-guard.ts — RiskGuard

4 段階チェック（全て通過で `allowed: true`）:

1. **日次損失キルスイッチ**: 当日の実弾損失合計が上限超過 → 全停止
2. **最大ポジション数**: 実弾 OPEN が上限到達 → 新規拒否
3. **異常レート検知**: 前回比乖離率が閾値超過 → 発注拒否
4. **ロット制限**: 超過時はクランプ（`adjustedLot`）

### 3.6 instruments.ts — 銘柄定義

**46 銘柄アクティブ**（FX4 + 円クロス3 + 株価指数6 + 暗号1 + コモディティ5 + 日本株19 + 米国株8）

`InstrumentConfig` の主要フィールド:
- `broker`: 'paper' | 'oanda' | 'alpaca'
- `tier`: 'A' | 'B' | 'C' | 'D'（ティア別ロット倍率で資金配分）
- `correlationGroup`: 相関グループ（同グループ同方向 2 件以上でブロック）
- `rateChangeTh`: フィルタ閾値
- `pnlMultiplier`: PnL 円換算倍率
- `tpSlMin` / `tpSlMax` / `rrMax`: サニティ上下限
- `trailingActivation` / `trailingDistance`: トレイリングストップ設定
- `assetClass`: 'forex' | 'cfd' | 'stock'（株式の場合）
- `stockSymbol` / `tradingHoursJST` / `tradingHoursET`: 株式用設定

### 3.7 filter.ts — フィルタ

**Ph.6 以降**: `shouldCallGemini()` は廃止。Logic エントリーが毎分実行されるため、AI 呼出頻度の制御は不要。

`isSkipSchedule()`: 重要指標発表時間帯の判定（news-trigger 側で参照予定）
- 米雇用統計（NFP）: 第1金曜 12:30〜14:30 UTC
- 米CPI: 第2火曜 12:30〜14:30 UTC
- 日銀会合: 該当月の第3〜4週 木金 02:00〜05:00 UTC
- 早朝禁止帯: 18:00〜22:00 UTC（JST 3:00〜7:00）

## 6. 注意事項

- **cron 予算制御**: 50s でタイムアウト。ニュース分析フォールバックも 35s で打切り
- **API キーローテーション**: AI 呼出は Path B + Param Review のみ。5 Gemini + 2 OpenAI + 1 Anthropic キーを巡回
- **D1 バッチ最適化**: スキップ銘柄の HOLD 記録、レート更新は `env.DB.batch()` で一括
- **PnL 単位**: 全て円換算（`pnlMultiplier` で変換）。S&P500: ×10、US10Y: ×5000 等
- **Path A 廃止（Ph.6）**: 毎分のAI銘柄判定を廃止。テクニカル判定は Logic が拡張パラメーターで自律実行
- **週末制約（⚠️ 変更禁止）**: FX市場クローズ中（金曜21:00〜日曜22:00 UTC）は暗号資産（BTC/ETH/SOL）のみ取引可。取引判断関数は `getTradeableInstruments(INSTRUMENTS, weekendStatus)` を経由すること（`src/weekend.ts`）。詳細は `CLAUDE.md §週末市場クローズ制約` を参照
- **テスト実行**: `npm test` で vitest が起動。`src/weekend.test.ts` が週末制約の回帰テスト（12件）を担う
