## 3. モジュール仕様

### 3.1 index.ts — エントリーポイント

**cron フロー**:
1. ワンタイムマイグレーション確認
2. `Promise.allSettled` で news / reddit / indicators / frankfurter を並列取得
3. ニュースソース統計を `news_fetch_log` にバッチ INSERT
4. 全銘柄の現在レートをマッピング（Yahoo 失敗時はキャッシュフォールバック）
5. 全ポジションの TP/SL チェック + トレイリングストップ
6. ニュース分析（Gemini → GPT → Claude、8s タイムアウト、予算制御付き）
7. 銘柄ごとにフィルタ → 候補リスト作成 → スコア降順ソート
8. スキップ銘柄をバッチ INSERT（HOLD 記録）
9. 上位 N 件を AI 判定（Gemini → GPT → Claude フォールバック）
10. BUY/SELL ならブローカー経由でポジション開設
11. 実行時間記録、30s 超過で WARN、ログパージ、日次サマリー

**HTTP ハンドラ**: 4 ルート（/, /api/status, /style.css, /app.js）

### 3.2 gemini.ts — AI API

**関数一覧**:

| 関数 | 用途 | タイムアウト |
|------|------|-------------|
| `getDecision()` | Gemini 銘柄判定 | 15s |
| `getDecisionGPT()` | GPT フォールバック | 15s |
| `getDecisionClaude()` | Claude フォールバック | 15s |
| `analyzeNews()` | Gemini ニュース分析 | 8s |
| `analyzeNewsGPT()` | GPT ニュース分析 | 8s |
| `analyzeNewsClaude()` | Claude ニュース分析 | 8s |

**プロンプト設計**: 銘柄情報（レート・指標・ニュース・Reddit・過去履歴・スパークライン）を構造化テキストで渡し、JSON レスポンス（`responseMimeType: 'application/json'`）を強制。

### 3.3 position.ts — ポジション管理

- `checkAndCloseAllPositions()`: 全 OPEN ポジションの TP/SL を一括チェック
- `openPosition()`: 新規ポジション作成（D1 INSERT）
- **トレイリングストップ**: `trailingActivation` 以上の含み益で SL を `trailingDistance` 幅に引上げ

### 3.4 broker.ts — ブローカー抽象化

**BrokerAdapter インターフェース**:
```typescript
interface BrokerAdapter {
  openPosition(params: OrderParams): Promise<BrokerResult>;
  closePosition(params: CloseParams): Promise<BrokerResult>;
  updateStopLoss(params: UpdateSLParams): Promise<BrokerResult>;
}
```

- **PaperBroker**: 何もしない（D1 記録は呼出元が行う）
- **OandaBroker**: OANDA REST API v20 で MARKET 注文（FOK）
- **getBroker()**: `TRADING_ENABLED` + 銘柄設定で自動ルーティング
- **withFallback()**: OANDA 失敗時にペーパーにフォールバック

### 3.5 risk-guard.ts — RiskGuard

4 段階チェック（全て通過で `allowed: true`）:

1. **日次損失キルスイッチ**: 当日の実弾損失合計が上限超過 → 全停止
2. **最大ポジション数**: 実弾 OPEN が上限到達 → 新規拒否
3. **異常レート検知**: 前回比乖離率が閾値超過 → 発注拒否
4. **ロット制限**: 超過時はクランプ（`adjustedLot`）

### 3.6 instruments.ts — 銘柄定義

17 銘柄の設定値（`InstrumentConfig`）:
- `broker`: 'paper' | 'oanda'
- `rateChangeTh`: フィルタ閾値
- `pnlMultiplier`: PnL 円換算倍率
- `trailingActivation` / `trailingDistance`: トレイリングストップ設定

### 3.7 filter.ts — フィルタ

`shouldCallGemini()`: 以下の条件で AI 呼出を判定
- レート変化が閾値以上
- 新規ニュースあり
- Reddit キーワード検出
- 前回 AI 呼出から一定時間経過（定期強制）
- 重要指標発表時間帯はスキップ

## 4. フロントエンド

### 4.1 構成

| ファイル | 配信パス | 役割 |
|----------|---------|------|
| dashboard.ts | GET / | HTML テンプレート（テンプレートリテラル） |
| app.js.ts | GET /app.js | クライアント JS（ES5 互換） |
| style.css.ts | GET /style.css | CSS（CSS 変数でテーマ対応） |

**制約**: Workers 上で直接配信するため、全てを TypeScript 文字列として管理。`export const JS = \`...\`;` の形式。

### 4.2 データフロー

```
app.js → fetch('/api/status') → レンダリング → 30秒ポーリング
```

### 4.3 主要関数（app.js.ts）

| 関数 | 役割 |
|------|------|
| `loadData()` | /api/status を fetch してレンダリング |
| `renderWatchlist()` | 銘柄一覧（保有中/待機中） |
| `renderPerfSummary()` | 最大DD・Sharpe・RR比・PF + 2σ有意性 |
| `renderEquityChart()` | SVG 資産推移グラフ |
| `drawPriceChart()` | SVG 24h 価格チャート（ボトムシート内） |
| `fmtYenCompact()` | 大数字の万/億表示 |

## 6. 注意事項

- **cron 予算制御**: 50s で AI 判定ループを打切り。ニュース分析フォールバックも 35s で打切り
- **API キーローテーション**: `_keyIndex` をインクリメントして 5 Gemini + 2 OpenAI キーを巡回
- **D1 バッチ最適化**: スキップ銘柄の HOLD 記録、レート更新は `env.DB.batch()` で一括
- **PnL 単位**: 全て円換算（`pnlMultiplier` で変換）。S&P500: ×10、US10Y: ×5000 等
