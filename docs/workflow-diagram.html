ロジックパラメーター UI — UX 心理学 7 原則
原則 1 / FR-701
Progressive Disclosure
Nielsen 2006 — 段階的開示
理論
一度に提示する情報量を最小化し、必要に応じて段階的に開示することで認知負荷（Cognitive Load）を低減する。
実装
param-card: 概要行（ペア名・バッジ・RR・残件数）のみ表示。タップで 6 項目詳細グリッドを展開。展開状態は localStorage で永続化。
初期画面の情報量を約 75% 削減。スクロール不要でカテゴリ全体を俯瞰できる。
原則 2 / FR-702
Goal Gradient Effect
Hull 1932 — 目標勾配効果
理論
ゴールに近づくほど行動への動機づけが強くなる（ラットの走行速度実験）。残量の可視化が緊迫感を生む。
実装
進捗バー: 残り 60%+ → 青（安心）、20〜60% → オレンジ（注意）、<20% → 赤 + urgent-pulse（緊迫）。
レビュートリガーまでの残り件数を感情的に可視化。20% 以下で自動的に緊張感を生成。
原則 3 / FR-703
Peak-End Rule
Kahneman 1999 — ピーク終端則
理論
体験の評価はピーク時と終端の印象で決まる。プロセス全体の平均ではなく、最も強い印象の瞬間が記憶に残る。
実装
レビュー履歴カード: AI 判断理由を最上段・青左ボーダー（.param-reason-block）で強調表示。変更内容・評価実績は下部に配置。
レビュー閲覧体験の記憶を「AI の良い判断」で塗りつぶす。否定的な数値も後続として認知される。
原則 4 / FR-704
Variable Reward
Skinner 1957 / Eyal 2014 Hook Model
理論
不規則な報酬スケジュール（変動比率強化）は固定報酬より強い行動強化を生む。SNS の「いいね」が典型例。
実装
UPGRADED バッジ: param_version ≥ 2 かつ last_reviewed_at が 12h 以内のみグローアニメーション（upgraded-glow 2s infinite）。
「銘柄が更新されているかも」という確認行動を習慣化。報酬（UPGRADED）は時間限定で希少性を演出。
原則 5 / FR-705
Weber–Fechner の法則
Weber 1834 / Fechner 1860 — 知覚の対数則
理論
知覚される変化量は実際の変化量の対数に比例する。大きな変化は大きなビジュアルで、小さな変化は控えめに表示する設計が自然。
実装
差分バッジ: |Δ| ≥ 15% → font-size: 12px / font-weight: 700 / 色付き。|Δ| < 15% → font-size: 11px / font-weight: 400 / グレー。
重要な変化（15%+）だけが自然に目立ち、ノイズとならない。視覚的階層が変化の重要性を伝える。
原則 6 / FR-706
チャンキング
Miller 1956 — マジカルナンバー 7±2
理論
短期記憶の容量は 7±2 チャンク。情報をグループ化（チャンキング）することで見かけの情報量を制限内に収める。
実装
カテゴリアコーディオン: 「為替 / 株価指数 / 債券 / 暗号資産 / コモディティ」でグループ化。ヘッダータップで折りたたみ、chevron 回転 0.25s。
17 銘柄を一度に表示せず 5 カテゴリに分割。開いているカテゴリの銘柄数を 7 以下に自然に制限。
原則 7 / FR-707
新奇性効果 + 希少性
Berlyne 1960 / Cialdini 2001 — 注意と希少性
理論
新奇な刺激（新しいもの）と希少性（限定時間）が注意を引き、即時行動を促す。スカーシティの原則。
実装
緊急ニュースバナー: EMERGENCY（スコア 90+）検出後 10 分以内のみ赤バナー表示（emergency-flash 1.4s）。時間経過で自動フェードアウト。
「10 分しか表示されない」という時間的希少性が即時確認行動を促進。過度な常時表示を避け信頼性を維持。
実装上のアクセシビリティ原則: @media (prefers-reduced-motion: reduce) では全アニメーション（urgent-pulse / upgraded-glow / emergency-flash）を animation: none に無効化。vestibular disorder（前庭障害）ユーザーへの配慮は UX 心理学適用の前提条件。全タッチターゲットは Apple HIG 準拠の min-height: 44px を維持。
