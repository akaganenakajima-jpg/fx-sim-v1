## 2. テーブル定義

### 2.1 positions（ポジション管理）

仮想/実弾ポジションのライフサイクルを管理する。

| カラム | 型 | NULL | デフォルト | 説明 |
|--------|------|------|-----------|------|
| id | INTEGER | NO | AUTOINCREMENT | 主キー |
| pair | TEXT | NO | 'USD/JPY' | 銘柄名 |
| direction | TEXT | NO | — | 'BUY' \| 'SELL' |
| entry_rate | REAL | NO | — | エントリーレート |
| tp_rate | REAL | YES | NULL | 利確レート（トレイリングで更新あり） |
| sl_rate | REAL | YES | NULL | 損切レート（トレイリングで更新あり） |
| lot | REAL | NO | 1.0 | ロットサイズ |
| status | TEXT | NO | 'OPEN' | 'OPEN' \| 'CLOSED' |
| pnl | REAL | YES | NULL | 損益（円換算、クローズ時に確定） |
| entry_at | TEXT | NO | — | エントリー日時（ISO 8601） |
| closed_at | TEXT | YES | NULL | クローズ日時（ISO 8601） |
| close_rate | REAL | YES | NULL | クローズレート |
| close_reason | TEXT | YES | NULL | 'TP' \| 'SL' \| 'TRAILING' \| 'MANUAL' |
| source | TEXT | YES | 'paper' | 'paper' \| 'oanda'（v2 追加） |
| oanda_trade_id | TEXT | YES | NULL | OANDA トレード ID（v2 追加） |
| realized_rr | REAL | YES | NULL | 実現リスクリワード比（クローズ時に確定） |

**PnL 計算式**:
```
BUY:  pnl = (close_rate - entry_rate) × pnlMultiplier
SELL: pnl = (entry_rate - close_rate) × pnlMultiplier

実現RR計算式:
BUY:  realized_rr = (close_rate - entry_rate) / (entry_rate - sl_rate)
SELL: realized_rr = (entry_rate - close_rate) / (sl_rate - entry_rate)

勝敗判定: realized_rr >= 1.0 → 勝ち（WIN）, < 1.0 → 負け（LOSE）
```

**ライフサイクル**:
```
OPEN → [TP到達] → CLOSED (close_reason='TP')
     → [SL到達] → CLOSED (close_reason='SL')
     → [トレイリングSL] → CLOSED (close_reason='TRAILING')
```

### 2.7 instrument_params（ロジックパラメーター）

銘柄ごとのロジックエントリーパラメーターを管理。AI レビューで更新される。

| カラム | 型 | NULL | デフォルト | 説明 |
|--------|------|------|-----------|------|
| pair | TEXT | NO | — | 主キー（銘柄名） |
| rsi_period | INTEGER | NO | 14 | RSI 計算期間 |
| rsi_oversold | REAL | NO | 30 | 売られすぎ閾値 |
| rsi_overbought | REAL | NO | 70 | 買われすぎ閾値 |
| adx_period | INTEGER | NO | 14 | ADX 計算期間 |
| adx_min | REAL | NO | 25 | エントリー最低 ADX 値 |
| atr_period | INTEGER | NO | 14 | ATR 計算期間 |
| atr_tp_multiplier | REAL | NO | 2.0 | ATR × 倍率 = TP 幅 |
| atr_sl_multiplier | REAL | NO | 1.0 | ATR × 倍率 = SL 幅 |
| vix_max | REAL | NO | 30 | VIX 上限（超過でスキップ） |
| regime_allow | TEXT | NO | 'TREND' | 許容レジーム（カンマ区切り） |
| review_trade_count | INTEGER | NO | 20 | レビュートリガー取引件数 |
| trades_since_review | INTEGER | NO | 0 | 前回レビュー後の取引数 |
| param_version | INTEGER | NO | 1 | パラメーターバージョン |
| reviewed_by | TEXT | YES | NULL | 最終レビュー主体（'AI' \| 'MANUAL'） |
| last_reviewed_at | TEXT | YES | NULL | 最終レビュー日時（ISO 8601） |
| prev_params_json | TEXT | YES | NULL | 前バージョンのパラメーター JSON |
| updated_at | TEXT | NO | — | 更新日時（ISO 8601） |
| vix_tp_scale | REAL | NO | 1.0 | VIX 警戒時（> vix_max×0.7）の TP 幅倍率 |
| vix_sl_scale | REAL | NO | 1.0 | VIX 警戒時（> vix_max×0.7）の SL 幅倍率 |
| strategy_primary | TEXT | NO | 'mean_reversion' | 優先戦略（'mean_reversion' \| 'trend_follow'） |
| min_signal_strength | REAL | NO | 0.0 | エントリー最低シグナル強度（0〜1） |
| macro_sl_scale | REAL | NO | 1.0 | マクロ下落時（S&P/日経 -1.5%超）の SL 幅倍率 |

**Ph.6 拡張パラメーター（v214 追加）**:

```
最終TP幅 = ATR × atr_tp_multiplier × vix_tp_scale（VIX > vix_max×0.7 時のみ適用）
最終SL幅 = ATR × atr_sl_multiplier × vix_sl_scale × macro_sl_scale（条件分岐）
エントリー可否 = signal_strength(RSI偏差+ER) ≥ min_signal_strength
戦略選択 = strategy_primary → trend_follow 時は ER ≥ adx_min/100 を追加チェック
```

### 2.9 news_trigger_log（ニューストリガーログ）

| カラム | 型 | NULL | デフォルト | 説明 |
|--------|------|------|-----------|------|
| id | INTEGER | NO | AUTOINCREMENT | 主キー |
| trigger_type | TEXT | NO | — | 'EMERGENCY' \| 'TREND_INFLUENCE' |

| news_score | INTEGER | NO | — | ニューススコア（1〜100） |

| detail | TEXT | YES | NULL | 詳細メッセージ |
| created_at | TEXT | NO | — | 記録日時（ISO 8601） |

**trigger_type 判定基準**:
- `EMERGENCY`: スコア ≥ 90 → PATH_B 強制発火フラグをセット
- `TREND_INFLUENCE`: スコア 70〜89 → 影響銘柄のパラメーターを一時調整
