# フロントエンド v7 — 設計意図書

## この文書の目的

モックアップ（mockup/full-app-v7.html）を本番に実装する際、
**なぜその要素がそこにあるのか**を理解するためのガイド。

見た目をコピーするのではなく、思想を実装する。

---

## 全体の思想

カスタマージャーニー設計書（docs/superpowers/specs/2026-03-24-ux-customer-journey-design.md）より:

ユーザーがアプリを開いた瞬間に4つの問いに答える:
1. 儲かっているか？（0.5秒）
2. なぜ？（3秒）
3. 何が起きている？（5秒）
4. 緊急はないか？（0秒 — 異常時のみ割り込み）

詳細タブは Phase 4（任意の深掘り）。各タブにも固有の「問い」がある。

---

## タブ1: 今 — 「儲かってる？なぜ？何が起きてる？」

### ヒーロー数字（-¥722）
- **意図**: 開いた瞬間の感情を決める。プラスなら安心、マイナスなら不安
- **重要**: 今日の損益が主役。残高は脇役。ROI/勝率/PFは補足情報
- **フォント**: 48px, weight 800。画面で最も目立つ要素
- **色**: プラス=緑、マイナス=赤（text-shadow で微妙なグロー）

### ストーリーカード
- **意図**: 「なぜ」を自然言語で3秒以内に伝える
- **青左ボーダー**: 「これは説明文です」という視覚的手がかり
- **ドライバーカード**: 利益TOP/損失TOPを並列表示。タップで学びタブへ遷移
- **チップ**: 影響要因を一目で把握。タップで詳細タブへ

### ニュース速報
- **意図**: 取引に影響したニュースを因果チェーンで見せる
- **構造**: ヘッドライン → AI判断 → 取引行動（BUY/SELL/影響なし）
- **タップ**: ニュースタブへ遷移（全30件を見る）
- **空の場合**: 「速報なし」ではなく、直近のニュース分析結果を表示すべき

### ポジションカード
- **意図**: 今持っているものと、それが勝っているか負けているかを一目で
- **左端色帯**: 勝ち=緑、負け=赤（::before疑似要素）
- **保有時間**: 8h超で⏰オレンジ（処分効果対策）
- **RR比**: 現在のリスクリワード
- **タップ**: ボトムシートでトレーサビリティ（なぜオープン/クローズ）

## タブ2: ニュース — 「何が起きた？どう影響した？」

### 核心思想
ただのニュースリストではない。**ニュース→AI判断→取引行動→結果**の因果チェーンを見せる。

### KPI（4つ）
- 取得件数 / AI分析済 / 取引発動 / 緊急
- **意図**: ニュースの「処理パイプライン」を可視化

### 取引に影響したニュース
- **意図**: 最も重要。「このニュースがこの取引を生んだ」という因果
- **構造**: バッジ(緊急/トレンド/情報) + ヘッドライン + AI判断テキスト + 取引行動
- **Why×5**: 折りたたみで「なぜBUY？→なぜscore=92？→なぜTP=$93.38？→...」
- **クロスリンク**: →今タブでポジション確認 / →AIタブで判定確認

### 分析済み・影響なし
- **意図**: AIが「影響なし」と判断したものも透明性のために表示
- 例: 日銀据え置き→「市場は織り込み済み」→影響なし

### 未分析ニュース
- **意図**: 処理されなかったニュースの存在を知らせる
- タイムスタンプ付きリスト + ソース別統計

## タブ3: 学び — 「何が効いて何が効かなかった？」

### 核心思想
KPIの羅列ではない。**パラメーター変更→結果のBefore/After**を見せる。
「AIがこう変えた→結果こうなった」の因果を検証する場。

### KPI（6つ）
- 勝率/PF/シャープ/最大DD/平均RR/総取引
- **意図**: 全体の健康状態を示すが、これは**前置き**

### エクイティカーブ
- **意図**: パラメーター変更マーカー（青点線）付きで、「変更後に改善したか」を視覚化
- **マーカー**: v2開始時点に縦線。その前後でカーブの傾きが変わったか

### 手法×環境マトリクス
- **意図**: mean_reversion vs trend_follow が、低VIX/中VIX/高VIX でどう違うかを表にして「どの戦略×環境の組み合わせが効いているか」を発見する

### 全体の結論
- **効いた変更 N件 / 効かなかった M件 / 判定中 K件**
- **意図**: 一言で「AIの最適化は機能しているか？」に答える

### 銘柄別 evo-card
- **意図**: 各銘柄について「何を変えて、結果どうなったか」をBefore/Afterで見せる
- **evo-verdict**: ✅改善中 / ❌悪化 / ⏳検証中
- **evo-chart**: ミニスパークライン + パラメーター変更マーカー
- **evo-change**: 具体的な変更内容 + 結果数値
- **Why×5**: なぜ効いた？/ なぜ効かなかった？を5段階で掘り下げ
- **クロスリンク**: →戦略で詳細 / →AIタブで判断を確認

## タブ4: AI — 「AIは正しかったか？」

### 核心思想
AIの行動の**アカウンタビリティ（説明責任）**。
全てのAI行動に**✅正解 / ❌不正解 / ⏳判定中**の判定をつける。

### ヒーロー正解率（68%）
- **意図**: 開いた瞬間に「AIは信頼できるか」がわかる
- **色**: 70%以上=緑、50-70%=オレンジ、50%未満=赤

### 種類別内訳
- ニュース分析: 72% / Param Review: 58%
- **意図**: どの種類のAI行動が正確で、どれが改善が必要かを分類

### Brierスコアスパークライン
- **意図**: 予測精度のトレンド。改善しているか悪化しているか

### verdict-card（判定カード）
- **意図**: 1つ1つのAI行動に「正解/不正解」の判定
- **正解の判定基準**:
  - Param Review: 変更後の勝率/RRが改善 → ✅正解
  - ニュース分析: BUY推奨→実際に上昇 → ✅正解
- **構造**: 行動内容 + AI理由 + 結果 + 判定 + Why×5 + クロスリンク

## タブ5: 戦略 — 「なぜ今のロジックはこうなった？」

### 核心思想
パラメーター表ではない。**各銘柄の進化の物語**。
「初期設定→AI変更→結果→現在」のストーリーを語る。

### journey-card
- **jc-summary**: 物語テキスト（例: 「初期設定から1回のAIレビューを経て、TP倍率とADX閾値を引き上げ。結果、勝率が48%→58%に改善。攻撃的なパラメーターが奏功。」）
- **スコア内訳バーグラフ**: RSI/ER/MTF/S/R/PA/BBの各スコアを横棒で可視化。「何がエントリーに寄与しているか」
- **タイムライン**: v1(初期)→v2(現在) の時系列。各ステップにAI理由+結果
- **パラメーター折りたたみ**: 生のパラメーター値は隠しておく（Progressive Disclosure）
- **クロスリンク**: →学びで成果確認

---

## タブ6: 系統 — 「壊れてないか？」

### 核心思想
全て正常なら**1秒で終わるタブ**。大きな✅とヘルスチェックだけ。
問題がある時だけ赤くなって詳細を見せる。

### ヘルスヒーロー
- **正常時**: 緑✅ + 「全システム正常」 → これだけ見ればOK
- **異常時**: 赤❌ + 「N件の問題を検出」

### DD段階バー
- **意図**: NORMAL→CAUTION→WARNING→HALT→STOP の5段階ゲージ
- 現在位置を「▲ 現在 1.8%」で示す

### ヘルスチェック6項目
- 全て緑なら詳細不要。異常がある項目だけ展開して詳細表示
- **タップで展開**: 内訳/24h推移/閾値

### ログ
- **異常のみ表示**（INFO は折りたたみ）
- **意図**: 正常時はログを見る必要がない。異常時のみ

## CSS表現の意図

### Liquid Glass タブバー
- **意図**: iOS 26の最新デザイン。浮遊感でコンテンツとナビゲーションの層を分離
- **border-radius:40px**: 完全ラウンド。モバイルの指に馴染む

### カード角丸24px
- **意図**: iOS 26標準。やわらかい印象。12pxは旧世代

### 青左ボーダー
- **意図**: 「説明テキスト」「AI判断」の視覚的手がかり。Apple Notes の引用ブロックと同じ

### 色の意味
- 緑(#30D158): 利益/改善/正解/正常
- 赤(#FF453A): 損失/悪化/不正解/異常
- オレンジ(#FF9F0A): 警告/注意/中間
- 青(#0A84FF): リンク/情報/判定中
- 紫(#BF5AF2): Param Review（AI知性の色）

### フォント階層
- 48px/800: ヒーロー数字（1つだけ）
- 17px/600: セクション内の重要値
- 15px/400: 本文
- 11px/600: ラベル・補足
