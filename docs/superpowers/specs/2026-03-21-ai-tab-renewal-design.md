# AI判断タブ フルリニューアル — デザイン仕様

**作成日**: 2026-03-21
**ステータス**: Approved (モックアップ審査完了)
**モックアップ**: `mockups/ai-tab-mixed-v2.html`

---

## 背景・課題

### コアペイン
現行のAI判断タブは「**判断基準・判断材料・判断内容・判断結果が揃っていない**」。
ユーザーが知りたい4要素が別々の場所に散在しており、1つの判断を評価するために画面をまたぐ必要がある。

### 具体的な不満
- 重要な判断が漏れる（ファーストビューで見えない）
- その判断が結果どうだったのかが分からない
- 判断材料（VIX・ニュース等）が見えない
- 判断基準（なぜその方向か）が記録されていない

---

## ゴール

| ゴール | 指標 |
|--------|------|
| 4要素を1カードで完結させる | 判断基準/判断内容/判断結果/時刻が1タップで確認可能 |
| 今日のパフォーマンスを即把握 | ファーストビューでKPI4指標が確認できる |
| なぜAIが動いたかを可視化 | 全判断にトリガー情報を付与 |

---

## デザイン方針（採用: Mix C+A+B）

3アプローチの良い部分を組み合わせたミックスデザイン：

- **C（KPIダッシュボード）**: ファーストビューに4指標カード
- **A（タイムライン）**: 左アクセントバーで結果を即識別できるカードリスト
- **B（4点詳細）**: タップで展開する4要素詳細パネル

---

## 画面構成

```
┌─────────────────────────────┐
│  ヘッダー                    │  AI判断 / 更新時刻 / アイコン2つ
├─────────────────────────────┤
│  KPIグリッド (2×2)           │  今日の判断 / AI的中率
│                             │  今日の損益 / 最新判断
├─────────────────────────────┤
│  トリガーカウント (3列)        │  ニュース起動 / レート変動 / 定期30m
├─────────────────────────────┤
│  判断タイムライン ヘッダー     │  「BUY/SELL のみ ›」フィルター付き
├─────────────────────────────┤
│  タイムラインカード（compact） │  × N件（BUY/SELL）
│    └ 詳細パネル（展開時）     │  4点グリッド + 推論 + 指標チップ
├─────────────────────────────┤
│  HOLD セパレーター            │  「HOLD X件 省略」
├─────────────────────────────┤
│  タイムラインカード（compact） │  × N件（HOLDより古いBUY/SELL）
├─────────────────────────────┤
│  タブバー                    │  資産 / AI判断(active) / 統計 / ログ
└─────────────────────────────┘
```

## コンポーネント仕様

### 1. ヘッダー

| 要素 | 仕様 |
|------|------|
| タイトル | 「AI判断」26px / font-weight: 700 |
| メタ情報 | 「HH:MM 更新 · N件分析済み」11px / label2 |
| アイコンボタン | 検索・更新 / 44×44px タッチターゲット |
| 背景 | rgba(28,28,30,0.95) + backdrop-filter: blur(20px) |

### 2. KPIグリッド（2×2）

4枚のカードで今日のサマリーを表示。アクセントバーは全カード同一の薄グレー（意味は値の色で伝える）。

| カード | 値の色 | 内容 |
|--------|-------|------|
| 今日の判断 | white（中立） | 件数 / BUY X · SELL Y · HOLD Z |
| AI的中率 | orange（精度） | X.X% / n=N · Brier X.XX |
| 今日の損益 | green/red（動的） | +¥X / X勝 Y敗 |
| 最新判断 | — | 方向バッジ / 銘柄 · X分前 |

**CSS**: `padding: 12px 16px` / `border-radius: 14px` / `gap: 8px`

### 3. トリガーカウントグリッド（3列）

今日のトリガー種別ごとの呼び出し回数。

| トリガー | 色 | 意味 |
|---------|-----|------|
| ニュース起動 | teal (#5ac8fa) | ニュースハッシュ変化による起動 |
| レート変動 | orange | ±0.05円超の変動による起動 |
| 定期 30m | gray (#aeaeb2) | 30分定期スキャン |

**CSS**: `border-radius: 12px` / `padding: 8px 12px` / `gap: 8px`

### 4. タイムラインカード（compact状態）

```
[左アクセントバー 3px] | [pair] [rate] [direction badge]    [time] [▸]
                       | [trigger badge] [reasoning 1行省略]
                       | [chip] [chip] [chip]           [● result]
```

**左アクセントバー色（結果に連動）**:
- `blue` = Open（保有中）
- `green` = TP（利確）
- `red` = SL（損切）

**方向バッジ**:
- BUY: `background: rgba(48,209,88,0.18)` / `color: #30d158`
- SELL: `background: rgba(255,69,58,0.18)` / `color: #ff453a`

**トリガーバッジ**:
- ニュース: teal系 border + bg
- レート変動: orange系 border + bg
- 定期 30m: gray系 border + bg

**CSS**: `border-radius: 14px` / `tl-inner padding: 12px` / `row gap: 8px` / `tl-chip: 11px`

### 5. 詳細パネル（タップで展開）

```
[trigger badge]  [trigger description text]
─────────────────────────────────────────
判断基準         判断内容
[value]          [DIRECTION · TP X / SL X]
判断結果         判断時刻/保有時間
[value]          [HH:MM:SS]
─────────────────────────────────────────
| [推論テキスト（引用スタイル）]
[chip] [chip] [chip] [chip]
```

**展開アニメーション**: `max-height` + `opacity` transition / `cubic-bezier(0.4,0,0.2,1)` / 300ms

**グリッド比**: `1fr 1.3fr`（右列に長いTP/SL値を収める）

**推論テキスト**: 左ボーダー2px（引用スタイル）/ ネストなし

### 6. HOLD セパレーター

```
────────────  HOLD X件 省略  ────────────
```

`margin: 8px 0` / `font-size: 11px` / `color: label3`

### 7. タブバー

| 状態 | color | font-weight |
|------|-------|-------------|
| active | --blue | 600 |
| inactive | --label2 | 400 |

`font-size: 11px` / `padding: 8px 0 24px`

## カラーシステム

### 意味的色定義（各色 = 1つの意味のみ）

| 色 | 変数 | 意味 |
|----|------|------|
| blue #0a84ff | `--blue` | 構造・Open状態・ナビゲーション |
| teal #5ac8fa | `--teal` | 情報・ニューストリガー |
| orange #ff9f0a | `--orange` | 精度指標・レートトリガー |
| green #30d158 | `--green` | 利益・TP・BUY |
| red #ff453a | `--red` | 損失・SL・SELL |
| gray #aeaeb2 | `--label3` | 中立・定期トリガー |

### 廃止した色
- **purple** (#bf5af2) — ニュースに使っていたが意味が不明確。tealに統合。

## UX設計根拠

### Progressive Disclosure（段階的開示）
- compact状態: pair/direction/trigger/result の4要素のみ → 認知負荷 最小
- 展開状態: 判断基準/判断内容/判断結果/時刻 + 推論 + 指標 → 必要な人だけ深掘り

### Variable Reward（変動報酬）
- タイムラインのカード色（青/緑/赤）が更新されるたびに新しい結果を通知

### Loss Aversion（損失回避）
- SL結果は赤アクセント + 赤テキストで明示。ただし「最新判断カード」はSELL方向でも青アクセント（方向 ≠ 損失）

### Serial Position Effect（系列位置効果）
- 最重要指標（今日の損益・最新判断）を KPI グリッドに先置き

## 実装スコープ

### 実装対象ファイル

| ファイル | 変更内容 |
|---------|---------|
| `src/dashboard.ts` | AI判断タブのHTML構造を全面書き換え |
| `src/style.css.ts` | AI判断タブ用CSSクラス追加（既存クラスは影響なし） |
| `src/app.js.ts` | AI判断タブのデータ描画ロジック書き換え |

### 実装しないこと
- バックエンドAPI変更なし（既存のdecisionsテーブルをそのまま使用）
- 他タブ（資産・統計・ログ）への変更なし
- 新規データ収集なし（トリガー種別はdecisionsテーブルの既存フィールドから判定）

### トリガー種別の判定ロジック
decisionsテーブルの既存カラムから推定：
- `reddit_signal` に値あり OR `news_summary` に変化 → `trigger-news`
- 前回レートとの差 ≥ 0.05 → `trigger-rate`
- 上記以外の定期実行 → `trigger-cron`
