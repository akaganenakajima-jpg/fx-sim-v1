## 1. 背景 — Ph.6 で何が変わったか

### 1.1 システムの性格の変化

| | Ph.5 以前 | Ph.6 以降 |
|---|---|---|
| AIの役割 | 毎分ハンドルを切る「運転手」 | たまにナビを更新する「アドバイザー」 |
| 判断の主体 | AI（Gemini → GPT → Claude） | Logic（RSI + ER + ATR + 拡張パラメーター） |
| ユーザーとAIの関係 | 「AIが何を考えているか」を監視 | 「ルールがどう機能しているか」を検証 |
| 変化の頻度 | 毎分：AI判定が変わる | 毎分：Logicは淡々と実行。変化はN件ごとのParam Review時 |
| 面白さの源泉 | 「AIが何と言ったか」（Variable Reward） | 「パラメーターを変えたら結果がどう変わったか」（実験→学習） |

**結論: ダッシュボードの主役が「AIの判断」から「ルールの成績」に移った。**

### 1.2 フロントエンドへの影響

#### 見せる意味がなくなった情報
- 毎分の銘柄別AIスコア（Path A 廃止）
- AI呼び出しフィルタ判定（`shouldCallGemini` 廃止）
- AI判定の動的上限制御
- 「AI判断」タブの旧コンテンツの大部分

#### 新たに必要になった情報
- Logic パラメーターの現在値と効果（因果関係）
- VIX/マクロ条件による動的 TP/SL 調整状態
- AI Param Review の提案内容と結果
- ニュースと銘柄の因果関係（Path B）
- 手法 × 環境マトリクス（戦略効果の可視化）

## 2. ペルソナ

### 2.1 ユーザー定義

**単一ユーザー（自分自身）** が3つの顔を持つ:

| 顔 | 関心事 | 頻度 |
|---|---|---|
| **投資家** | 儲かっているか。リスクは許容範囲か | 毎回 |
| **開発者** | システムは正常か。バグはないか | 異常時 + 定期 |
| **学習者** | パラメーターと結果の因果関係。何が効いて何が効かなかったか | 振り返り時 |

### 2.2 核心的なインサイト

3つのモードは「切り替える」のではなく、**1画面で同時に満たされる**。

ダッシュボードを開いた瞬間に知りたいことは:

1. **儲かっているか？**（結果）
2. **なぜ儲かっている/損しているか？**（因果）
3. **今、何が起きているか？**（状況）
4. **緊急性の高い事柄はないか？**（警報）

この4つが **一目で** わかることが最重要要件。

## 3. カスタマージャーニーマップ

### 3.1 ジャーニーフロー

```
開く（0秒）
  │
  ▼
Phase 1: 結果 + 警報（0.5秒）━━━━━━━━━━━━━━━━━━━
  「儲かってる？ 異常はない？」
  ├─ PnL ヒーロー（残高・今日損益・ROI）
  ├─ 異常バナー（DD警告・緊急ニュース・システムエラー）← 無ければ不可視
  └─ 判定: 安心 or 不安 → 不安なら Phase 2 へ強く引かれる
  │
  ▼
Phase 2: 因果サマリー（3秒）━━━━━━━━━━━━━━━━━━━━━
  「なぜこうなっている？」
  ├─ ナラティブ: 自然言語で今日の状況を要約
  │   例: 「Goldが+420円（ATR TP到達）、VIX=28でSL縮小中」
  ├─ キードライバー: 利益Top / 損失Top / 最大影響要因
  │   例: 「利益: Gold TP +420 / 損失: Nikkei SL -225 / 要因: VIX=28」
  ├─ ビジュアル: 銘柄×要因ヒートマップ（コンパクト版）
  └─ 判定: 納得 or 「もっと知りたい」→ 詳細タブへ
  │
  ▼
Phase 3: 状況俯瞰（5秒）━━━━━━━━━━━━━━━━━━━━━━
  「今、市場で何が起きている？」
  ├─ 銘柄ウォッチリスト（保有中 + スパークライン）
  ├─ 市場指標サマリー（VIX・米10Y・日経・S&P）
  └─ ニュースモーダル（下からスワイプ）← 現行UXを維持
  │
  ▼
Phase 4: 深掘り（任意）━━━━━━━━━━━━━━━━━━━━━━━
  「詳しく知りたい」→ タブで遷移
  ├─ パラメーター詳細（戦略タブ）
  │   銘柄別の現在値・AI Review 履歴・変更差分
  ├─ AI・ニュース詳細（AI・ニュースタブ）
  │   Path B 分析結果・ニュースと銘柄の因果関係
  ├─ 統計・学習（統計タブ）
  │   勝率・PF・手法×環境マトリクス・エクイティカーブ
  └─ システム（ログタブ）
      ログ・RiskGuard・cron タイミング
```

### 3.2 感情曲線

```
😊 安心     ──────╮                    ╭──── 納得・学び
                   ╰─ 結果を見る ──╮    │
😐 ニュートラル                     │    │
                                   ╰─ 因果を読む ─╯
😟 不安     ── 異常バナー割込 ──→ 即座に原因特定 ──→ 😊 安心
```

| 朝起きて開く | Phase 1-2（結果 + 因果サマリー） | 浅い |

### 4.2 タブ構成の再編

**並び順の原則**: 投資家 → 学習者 → 開発者の順。ユーザーの最頻モードが左、開発者向けは最後尾。

| # | 現タブ名 | 新タブ名 | 対象モード | 役割の変化 |
|---|---|---|---|---|
| 1 | 資産 (tab-portfolio) | **ダッシュボード** | 投資家 | Phase 1-3 のホーム。因果サマリーを新設 |
| 2 | 統計 (tab-stats) | **統計** | 投資家+学習者 | エクイティカーブ・手法マトリクス強化 |
| 3 | AI判断 (tab-ai) | **AI・ニュース** | 学習者 | Path B + Param Review + ニュース因果に焦点 |
| 4 | 戦略 (tab-strategy) | **戦略** | 学習者 | パラメーター管理が主役。Logic の設定と効果 |
| 5 | ログ (tab-log) | **システム** | 開発者 | RiskGuard + システムログ + cron タイミング |

**HTML id 属性の方針**: 内部IDは変更しない（`tab-portfolio`, `tab-ai` 等を維持）。ラベルのみ変更する。理由: IDの変更はフロントエンド全体（`switchTab()`, `localStorage` キー, CSS セレクタ）に波及するため、リスクに見合わない。

### 4.3 因果サマリーの3層構造

因果サマリーは Ph.6 ダッシュボードの **新しい核**。3つの表現形式を併置する:

#### A. ナラティブ（自然言語要約）
- バックエンドで生成する1-2文のテキスト
- 今日のPnLの主要因（Top銘柄 + 理由）を自然言語で記述
- 例: 「Gold が ATR×3 の TP 到達で +420円。VIX=28 のため SL 幅を 0.8 倍に縮小中」
- AI を使わずロジックで生成（テンプレート + 条件分岐）
- **最大100文字**（モバイル1カラムでの画面圧迫を防止）

#### B. キードライバー（構造化要因リスト）
- 利益 Top / 損失 Top の銘柄と金額
- 最大影響要因（VIX 水準、マクロ下落、Param Review 変更）
- 数字 + ラベルの簡潔な表示

#### C. ビジュアル（ヒートマップ）
- 銘柄 × 要因のコンパクトなビジュアライゼーション
- 色の濃淡で「どの銘柄の何が効いているか」を直感的に把握
- **実装方式**: CSS Grid + 背景色（外部ライブラリ不使用、Workers バンドルサイズ維持）
- タップで詳細タブへ遷移（FR-711 ディープリンク）

### 4.4 因果サマリーの API 仕様

`/api/status` の `StatusResponse` に以下のフィールドを追加:

```typescript
// StatusResponse に追加
causalSummary: {
  // A. ナラティブ
  narrative: string;  // テンプレート生成の1-2文（例:「Goldが+420円…」）

  // B. キードライバー
  drivers: {
    profitTop:  { pair: string; pnl: number; reason: string } | null;
    lossTop:    { pair: string; pnl: number; reason: string } | null;
    factors: Array<{
      type: 'vix' | 'macro' | 'param_review' | 'news' | 'trailing' | 'delisted';
      label: string;   // 「VIX=28 → SL縮小0.8×」
      severity: 'high' | 'medium' | 'low';
    }>;
  };

  // C. ヒートマップ
  heatmap: Array<{
    pair: string;
    pnlToday: number;
    factors: Record<string, number>;  // { vix: -0.2, atr_tp: 0.8, ... }
  }>;
} | null;
```

**ナラティブ生成ロジック**（バックエンド `api.ts` 内、AIは使わない）:

| 条件 | 生成テンプレート |
|---|---|
| 利益Top銘柄あり + TP決済 | 「{pair}が+{pnl}円（{reason}）。」 |
| VIX > vix_max×0.7 | 「VIX={vix}のためSL幅を{scale}倍に縮小中。」 |
| マクロ下落（S&P/日経 -1.5%超） | 「{index}が{change}%下落、SL幅{scale}倍に縮小。」 |
| Param Review 実行あり（12h以内） | 「Param Reviewが{pair}の{param}を{old}→{new}に変更提案。」 |
| ポジション0件 + 当日取引なし | 「本日はまだ取引がありません。」 |
| データ取得失敗 | `null`（フロントエンドはセクション非表示） |

**ナラティブ生成に必要なDBクエリ**（`getApiStatus()` 内で既存クエリと並列実行）:

| # | クエリ | 用途 |
|---|---|---|
| 1 | `SELECT pair, pnl, close_reason FROM positions WHERE status='CLOSED' AND DATE(closed_at)=DATE('now')` | 当日確定銘柄の利益Top/損失Top |
| 2 | `SELECT pair, direction, entry_rate FROM positions WHERE status='OPEN'` | 含み損益の計算（既存クエリ流用） |
| 3 | `SELECT key, value FROM market_cache WHERE key IN ('vix','sp500_change','nikkei_change')` | VIX/マクロ要因の判定 |
| 4 | `SELECT pair, trigger_type, score FROM news_trigger_log WHERE created_at > datetime('now','-12 hours') ORDER BY created_at DESC LIMIT 5` | 直近のニューストリガー |
| 5 | `SELECT pair, changed_params FROM param_review_log WHERE created_at > datetime('now','-12 hours') ORDER BY created_at DESC LIMIT 3` | 直近のParam Review変更 |

**ヒートマップの「要因」軸の定義**:

| 要因キー | 意味 | 値の範囲 |
|---|---|---|
| `pnl_closed` | 当日の確定損益 | 円（正:利益/負:損失） |
| `pnl_unrealized` | 含み損益 | 円 |
| `vix_effect` | VIX による TP/SL スケール影響度 | 0〜1（0=影響なし, 1=最大縮小） |
| `param_changed` | Param Review 変更有無 | 0 or 1 |
| `news_impact` | Path B ニュース影響 | 0〜100（ニューススコア） |

### 4.5 警報バナーのトリガー条件

| 警報種別 | トリガー条件 | データソース | 表示色 |
|---|---|---|---|
| DD 警告 | `riskStatus.killSwitchActive === true` | `api.ts` riskStatus | 赤 |
| DD 注意 | `riskStatus.todayLoss / riskStatus.maxDailyLoss > 0.8` | riskStatus | オレンジ |
| 緊急ニュース | `newsAnalysis` に `attention === true` かつ 10分以内 | newsAnalysis | 赤 |
| システムエラー | `systemLogs` 直近5件に `level === 'ERROR'` が含まれる | systemLogs | オレンジ |
| 孤立ポジション | `causalSummary.factors` に `type === 'delisted'` | causalSummary | 黄 |

表示ルール:
- 複数の警報が同時発生した場合は **重篤度順**（赤 → オレンジ → 黄）にスタック表示
- 正常時はバナー領域を **完全に非表示**（高さ0）— 画面を占有しない

### 4.6 ディープリンク仕様（FR-711）

因果サマリーの各要素をタップすると、対応するタブの該当セクションにジャンプする。

```typescript
// switchTab() を拡張して、タブ切替 + スクロール先を指定可能にする
function switchTab(tabId: string, scrollTo?: string): void {
  // 1. タブ切替（既存ロジック）
  // 2. scrollTo が指定されていれば、該当セクションまでスクロール
  //    document.getElementById(scrollTo)?.scrollIntoView({ behavior: 'smooth' })
  // 3. 該当セクションを一時的にハイライト（0.5秒間 背景色を薄く点灯）
}
```

| タップ元（因果サマリー） | 遷移先タブ | スクロール先 |
|---|---|---|
| 利益Top / 損失Top 銘柄 | 統計 | `#perf-{pair}` 銘柄別パフォーマンス |
| VIX / マクロ要因 | 戦略 | `#params-{pair}` パラメーターカード |
| Param Review 変更 | 戦略 | `#review-history` 変更履歴 |
| ニュース影響 | AI・ニュース | `#news-analysis` ニュース分析 |

## 5. UX 原則（Ph.6 対応）

### 5.1 既存の7原則との整合

| ID | 原則 | Ph.6 での適用 |
|---|---|---|
| FR-701 | Progressive Disclosure | 因果サマリー（概要）→ タブ（詳細）の2段階に適用 |
| FR-702 | Goal Gradient | Param Review の進捗バー（残り取引件数/目標）を維持 |
| FR-703 | Peak-End 則 | TP 決済時の祝福バナー + 因果ナラティブを最上段表示 |
| FR-704 | Variable Reward | Param Review 後の UPGRADED バッジ（AI提案の意外性） |
| FR-705 | 差分バッジサイズ分岐 | パラメーター変化量に応じた視覚的強弱を維持 |
| FR-706 | カテゴリアコーディオン | ウォッチリストのカテゴリ折りたたみを維持 |
| FR-707 | 緊急ニュースバナー | Phase 1 の警報バナーとして統合 |

### 5.2 新たに追加する原則

| ID | 原則 | 内容 |
|---|---|---|
| FR-708 | Glanceable Design | ダッシュボード Phase 1-3 は一目で4層を伝える。情報密度より視認性 |
| FR-709 | Causal Narrative | PnL の数字だけでなく「なぜ」を常にセットで提示する |
| FR-710 | Alert-Only Interruption | 警報は異常時のみ表示。正常時は画面を占有しない |
| FR-711 | Drill-Down Consistency | サマリーの各要素をタップすると、対応する詳細タブの該当箇所にジャンプ |

## 8. 実装優先度（フェーズ分け）

| Phase | 内容 | 依存 |
|---|---|---|
| **Step 1** | タブリネーム + 並び順変更（開発者→最後尾）+ 不要コンポーネント削除（AI期待ランキング等） | なし |
| **Step 2** | 因果サマリー（ナラティブ + キードライバー）— テキストベースのみ | API 拡張（`causalSummary`フィールド追加） |
| **Step 3** | 警報バナー統合（Phase 1 に集約、正常時非表示） | なし |
| **Step 4** | 因果サマリー（ヒートマップ）— CSS Grid ビジュアル | Step 2 の API データ |
| **Step 5** | ディープリンク（FR-711: サマリー → 詳細タブジャンプ） | Step 2 + タブ内セクションIDの整備 |

**最小実装（Step 1-3）で「一目で4層が分かる」体験を達成可能。** Step 4-5 は段階的に追加。

## 9. エラー状態・フォールバック表示

| 状態 | 表示内容 |
|---|---|
| ポジション0件 + 当日取引なし | ナラティブ:「本日はまだ取引がありません。」キードライバー: 空欄 |
| `causalSummary` が `null`（API エラー） | 因果サマリーセクション全体を非表示。Phase 1（結果ヒーロー）+ Phase 3（状況）のみ表示 |
| ヒートマップデータが空 | ヒートマップセクションのみ非表示（ナラティブ + キードライバーは表示） |
| 市場レート取得失敗 | 該当銘柄のスパークラインをグレーアウト + 「取得失敗」ラベル |
| 全データ取得失敗 | 「データを取得できませんでした。再読み込みしてください」+ リトライボタン |

## 10. レスポンシブ対応
