## 1. 機能要件

### 1.1 データ収集（FR-100）

| ID | 機能 | 詳細 |
|----|------|------|
| FR-101 | レート取得 | Yahoo Finance API で 17 銘柄の現在価格を 1 分ごとに取得。失敗時は frankfurter.app（USD/JPY のみ）にフォールバック |
| FR-102 | ニュース取得 | NHK, Yahoo ファイナンス, Reuters, WSJ, Kabutan, Google News 等から RSS/JSON で取得。最大 12 件を AI 分析に渡す |
| FR-103 | Reddit 取得 | r/Forex の新着投稿から FX 関連キーワードを検出 |
| FR-104 | 市場指標 | VIX, 米10年債, 日経225, S&P500 等を Yahoo Finance から一括取得 |
| FR-105 | キャッシュ | Yahoo 障害時は market_cache テーブルの前回レートでフォールバック |

### 1.2 AI 判定（FR-200）

| ID | 機能 | 詳細 |
|----|------|------|
| FR-201 | Gemini 判定 | Gemini 3.1 Pro Preview に銘柄情報を渡し BUY/SELL/HOLD を取得 |
| FR-202 | フォールバック | Gemini 429 時は GPT-4.1 mini → Claude Sonnet 4 に順次フォールバック |
| FR-203 | ニュース分析 | 新規ニュース検出時に AI でマーケットインパクトを分析（注目フラグ付与） |
| FR-204 | フィルタ | レート変化率・新ニュース・Reddit キーワード・時間帯で AI 呼出を判定 |
| FR-205 | 優先度スコア | ボラティリティ + 市場セッション時間帯ボーナスで銘柄を優先順位付け |
| FR-206 | 動的上限 | 前回実行時間に応じて AI 呼出数の上限を動的調整（3〜10 件） |
| FR-207 | クールダウン | Gemini 429 後は一定時間 AI 呼出をスキップ |
| FR-208 | API キーローテーション | Gemini × 5 キー、OpenAI × 2 キーを巡回使用 |

### 1.3 ポジション管理（FR-300）

| ID | 機能 | 詳細 |
|----|------|------|
| FR-301 | ポジション開設 | BUY/SELL 判定時に D1 に OPEN レコードを作成 |
| FR-302 | TP/SL 自動決済 | 1 分ごとに全 OPEN ポジションの TP/SL を現在レートと比較 |
| FR-303 | トレイリングストップ | 利益が一定値に達したら SL を現在値に追従（銘柄ごとに設定） |
| FR-304 | 銘柄別最大1件 | 同一銘柄に複数ポジションを同時に持たない |
| FR-305 | PnL 計算 | `(close - entry) × pnlMultiplier`（方向により符号反転）で円換算 |

### 1.4 実弾取引（FR-400）

| ID | 機能 | 詳細 |
|----|------|------|
| FR-401 | ブローカー抽象化 | BrokerAdapter インターフェースで Paper/OANDA を切替 |
| FR-402 | OANDA 発注 | REST API v20 経由で MARKET 注文（FOK） |
| FR-403 | RiskGuard | 日次損失キルスイッチ・最大ポジション数・ロット制限・異常レート検知 |
| FR-404 | フォールバック | OANDA 失敗時はペーパートレードにフォールバック |

### 1.5 ダッシュボード（FR-500）

| ID | 機能 | 詳細 |
|----|------|------|
| FR-501 | 5 タブ構成 | 資産 / AI 判断 / 統計 / ログ / 戦略 |
| FR-502 | PnL ヒーロー | 資産残高・損益・ROI・勝率・取引数をリアルタイム表示 |
| FR-503 | ウォッチリスト | 保有中/待機中の銘柄を一覧表示（スパークライン付き） |
| FR-504 | ボトムシート | 銘柄タップで詳細情報 + 24h チャート表示 |
| FR-505 | ニュースドロワー | 下からスワイプでニュース一覧（AI 分析バッジ付き） |
| FR-506 | 資産推移グラフ | SVG で直近 30 件の資産推移を描画 |
| FR-507 | 2σ 有意性表示 | 勝率・PF の統計的有意性を表示 |
| FR-508 | テーマ切替 | ダーク/ライトモード手動切替 |

### 1.6 ロジックパラメーター管理（FR-600）

| ID | 機能 | 詳細 |
|----|------|------|
| FR-601 | 銘柄別パラメーター保持 | `instrument_params` テーブルで 17 銘柄ごとの RSI/ADX/ATR パラメーターを管理 |
| FR-602 | AI パラメーターレビュー | 前回レビュー後 N 件取引到達時、AI が勝率/RR/PF を評価し最適パラメーターを提案 |
| FR-603 | レビュー履歴記録 | `param_review_log` に変更前後の JSON・AI 判断理由・評価実績を記録 |
| FR-604 | バージョン管理 | `param_version` でパラメーターの世代を追跡し、前バージョンを `prev_params_json` に保持 |
| FR-605 | ニューストリガー | 緊急ニュース（スコア 90 以上）検出時に PATH_B 強制発火フラグを設定 |
| FR-606 | トレンドインフルエンス | スコア 70〜89 のニュース検出時に影響銘柄のパラメーターを一時調整 |
| FR-607 | ニュースログ記録 | `news_trigger_log` に trigger_type / 銘柄 / スコア / 詳細を記録 |

### 1.7 UX / フロントエンド要件（FR-700）

> Apple HIG + UX 心理学（認知負荷軽減・目標勾配・Peak-End・変動型報酬）に基づく 7 原則

| ID | 原則 | 詳細 |
|----|------|------|
| FR-701 | Progressive Disclosure | パラメーターカードを概要行のみ表示し、タップで詳細 6 項目グリッドを展開。展開状態は `localStorage` で永続化 |
| FR-702 | Goal Gradient | 進捗バー（残り取引件数/目標件数）を青→オレンジ→赤でカラー変化。残り 20% 以下は `urgent-pulse` アニメーション |
| FR-703 | Peak-End 則 | レビュー履歴でAI判断理由を最上段・青左ボーダー（`.param-reason-block`）で強調表示 |
| FR-704 | Variable Reward | `param_version ≥ 2` かつ 12h 以内のレビュー済み銘柄に `UPGRADED` グローアニメーションバッジ |
| FR-705 | 差分バッジサイズ分岐 | パラメーター変化量 ≥ 15% を太字・大サイズ（12px / 700）で表示、< 15% は標準サイズ（11px / 400） |
| FR-706 | カテゴリアコーディオン | カテゴリヘッダータップで銘柄リストを折りたたみ。chevron rotate transition 0.25s |
| FR-707 | 緊急ニュースバナー | EMERGENCY 検出後 10 分以内に赤バナー（`#emergency-news-banner`）を表示し、残り時間経過後に自動フェードアウト |

## 2. 非機能要件

### 2.1 性能

| ID | 要件 | 基準 |
|----|------|------|
| NFR-01 | cron 実行時間 | 平均 < 15s、最大 < 50s |
| NFR-02 | ダッシュボード応答 | < 500ms（/api/status） |
| NFR-03 | AI API タイムアウト | 15s（判定）/ 8s（ニュース分析） |
| NFR-04 | Yahoo Finance タイムアウト | 5s（18 並列） |

### 2.2 可用性

| ID | 要件 | 基準 |
|----|------|------|
| NFR-05 | システム稼働率 | > 99%（Cloudflare SLA 準拠） |
| NFR-06 | データソース冗長性 | Yahoo 障害時キャッシュフォールバック |
| NFR-07 | AI 冗長性 | 3 プロバイダーフォールバック |

### 2.3 セキュリティ

| ID | 要件 | 基準 |
|----|------|------|
| NFR-08 | API キー管理 | Cloudflare Secrets で管理。コード内ハードコード禁止 |
| NFR-09 | RiskGuard | 実弾取引に 4 段階の安全装置 |
| NFR-10 | 入力検証 | 外部 API レスポンスの型チェック |

### 2.4 運用

| ID | 要件 | 基準 |
|----|------|------|
| NFR-11 | ログ | system_logs テーブルに INFO/WARN/ERROR を記録（500 件ローテーション） |
| NFR-12 | 日次サマリー | JST 0:00 に自動記録 |
| NFR-13 | ログパージ | 500 件超は古いものから自動削除 |
