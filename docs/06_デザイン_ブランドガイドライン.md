## 4. レイアウト

### 4.1 グリッドシステム

- **基本単位**: 8pt グリッド（4pt 許容）
- **画面幅**: 375px（最小）〜 428px（iPhone Pro Max）
- **セーフエリア**: `env(safe-area-inset-*)` 対応
- **カード padding**: 16px（12px 許容。**14px / 10px / 9px 禁止** — 4pt非倍数）
- **カード間 gap**: 8px
- **角丸**: メインカード **16px**、バッジ 6〜8px、ボタン 22px（カプセル）
  - 同一階層の要素は必ず同じ border-radius を使用すること
  - 旧値 12px → 16px に統一済み（2026-03-21 審査で修正）

### 4.2 画面構成

```
┌──────────────────────────┐
│ ヘッダー (safe-area + ~62px)│ タイトル + 更新時刻 + ボタン
├──────────────────────────┤
│ コンテンツエリア           │ タブごとに切替
│ padding-top:             │
│  calc(safe-area + 74px)  │ ← ヘッダー実高さ(62px) + 余白(12px)
│  ※ 58px では不足（修正済）│
│  ┌────────────────────┐  │
│  │ カード              │  │ padding: 16px, radius: 16px
│  └────────────────────┘  │
│                          │
├──────────────────────────┤
│ タブバー (49px)           │ 4タブ + safe-area-bottom
├──────────────────────────┤
│ ニュースドロワー           │ 下からスワイプ展開
└──────────────────────────┘
```

**コンテンツ余白の計算式**:
- ヘッダー高さ = `env(safe-area-inset-top)` + padding-top(12px) + タイトル行高(~38px) + padding-bottom(12px) ≈ safe-area + 62px
- コンテンツ padding-top = safe-area + **74px**（= ヘッダー 62px + 余白 12px）
- `58px` は不足 → ヘッダーとコンテンツが重なる

```css
  50%       { opacity: 0.55; }

/* 変動型報酬 — UPGRADED バッジのグローアニメーション */
@keyframes upgraded-glow {
```

## 9. UX 心理学の適用

### 9.1 既存 UI への適用

| 法則 | 適用箇所 |
|------|----------|
| Peak-End Rule | 資産推移グラフの最終値を強調表示 |
| Variable Reward | 30s ごとのデータ更新で変化を提供 |
| Goal Gradient | TP ラインまでの進捗をスパークラインで可視化 |
| Loss Aversion | 損失は赤色で控えめに、利益は緑色で大きく表示 |
| Serial Position | 最重要情報（残高）を最上部に配置 |
| Cognitive Load | 5 タブで情報を分散、1 画面あたり 7±2 要素 |
| Zeigarnik Effect | 統計的有意性進捗バーで「未完了感」を演出 |

### 9.2 ロジックパラメーター UI への適用（Ph.3〜5）

> 学術的根拠付きの 7 原則実装仕様。変更時は理論的根拠を維持すること。

| 原則 | 学術的根拠 | UI 実装 | 期待効果 |
|------|-----------|---------|---------|
| **Progressive Disclosure** | Nielsen 2006 — 段階的開示で認知負荷を低減 | param-card: 概要行のみ → タップで詳細展開 | 初期画面の情報量を 1/4 に削減 |
| **Goal Gradient** | Hull 1932 — ゴール近接でモチベーション上昇 | 進捗バー: 残り 60%→青, 20%→オレンジ, <20%→赤点滅 | レビュートリガーまでの残り件数を感情的に可視化 |
| **Peak-End 則** | Kahneman 1999 — 体験評価はピーク+終端で決まる | レビュー履歴: AI 理由を最上段・青ボーダーで強調 | レビュー体験の印象を最良の情報（AI 判断）で形成 |
| **Variable Reward** | Skinner 1957 / Eyal 2014 Hook Model | UPGRADED バッジ: 12h 以内のみグローアニメーション | 定期確認行動を強化（不規則な報酬スケジュール） |
| **差分バッジサイズ分岐** | Weber–Fechner の法則 — 知覚は変化量に比例 | ≥ 15% 変化: 12px / 700。< 15%: 11px / 400 | 重要な変化だけが視覚的に際立つ |
| **カテゴリアコーディオン** | Miller 1956 — チャンキングで 7±2 の制限を克服 | カテゴリヘッダーで銘柄群を折りたたみ | 一度に表示する銘柄数をカテゴリ単位で管理 |
| **緊急ニュースバナー** | Berlyne 1960 — 新奇性と強度が注意を引く | EMERGENCY 10 分以内: 赤バナー自動表示→自動消滅 | 時間制限（scarcity）が即時行動を促進 |

**実装注意事項**:
- `@media (prefers-reduced-motion: reduce)` では全アニメーションを `none` にする（アクセシビリティ優先）
- タッチターゲットは必ず `min-height: 44px`（Apple HIG § Ergonomics）
- `localStorage` キー: `param-expanded-cards`（展開状態）、`param-collapsed-cats`（カテゴリ折りたたみ）

## 10. HIG & UX 監査 — 恒久禁止ルール

> **発効日**: 2026-03-21
> UX心理学 + Apple HIG 精密審査（ソース & ブラウザ視覚両面）で発見した違反。
> フロントエンド開発・改修時は必ずこのセクションを確認すること。

### 10.1 フォントサイズ（H2 違反）

| ルール | 理由 |
|--------|------|
| **最小フォントサイズは 11px（Caption2）** | 9px・10px はモバイルで判読不能。WCAG 最低基準未達 |
| ラベル・サブテキスト: 最小 11px | `hero-stat-label`, `market-band-label`, `stat-item-label`, `stat-item-sub` で 9px を使用して違反 |
| タブバーラベルのみ 10px 例外許容 | iOS/iPadOS システム標準に合わせた唯一の例外 |

```css
/* ❌ 禁止 */
font-size: 9px;
font-size: 10px;  /* タブバー以外 */

/* ✅ 正しい */
font-size: 11px;  /* Caption2 — ラベル・サブテキストの最小値 */
font-size: 12px;  /* Caption1 */
font-size: 13px;  /* Footnote */
```

### 10.2 カラーコントラスト（H5 違反 — WCAG AA）

| ルール | 理由 |
|--------|------|
| **`#636366` をテキストに使用禁止** | ダーク背景での対比比 3.96:1 → WCAG AA（4.5:1）未達 |
| セカンダリテキストは `#8e8e93` を最低値とする | 対比比 6.6:1 → AA 適合 |
| テーシャリテキストは `#8e8e93` （同上） | `rgba(235,235,245,0.3)` は使用箇所を限定する |

```css
/* ❌ 禁止 */
color: #636366;  /* 対比比 3.96:1 — WCAG AA 未達 */

/* ✅ 正しい */
color: #8e8e93;  /* 対比比 6.6:1 — WCAG AA 適合 */
```

### 10.3 タッチターゲット（H1 違反）

| ルール | 理由 |
|--------|------|
| **全インタラクティブ要素に `min-height: 44px`** | Apple HIG 必須要件 |
| リストの行（ランキング・ペア行）: `min-height: 44px` | padding だけでは不十分な場合がある |
| トグル/切替ボタン: `min-height: 44px` | 視覚的に小さく見えても最低 44px 必要 |

```css
/* ✅ 正しい */
.interactive-row {
  min-height: 44px;
  display: flex;
  align-items: center;
}
```

### 10.4 8ptグリッド（H4 違反）

| 禁止値 | 置換先 | 理由 |
|--------|--------|------|
| `14px` | `12px` または `16px` | 4pt非倍数 |
| `10px` | `8px` または `12px` | 4pt非倍数 |
| `9px`  | `8px` | 4pt非倍数 |

```css
/* ❌ 禁止 */
padding: 9px 14px;
padding: 10px 14px;

/* ✅ 正しい */
padding: 12px 16px;
padding: 8px 12px;
```

### 10.5 角丸の一貫性（H7 違反）

| ルール | 理由 |
|--------|------|
| **同一階層のカードは全て `border-radius: 16px`** | 12px と 16px が混在し、品質が低く見える |
| バッジ・ピル: `border-radius: 6px` 〜 `8px` | 小要素はより小さい radius（concentric rule） |
| 異なる値を混在させない | `border-radius: 10px` と `6px` の混在は禁止 |

### 10.6 `:active` フィードバック（I1 違反）

| ルール | 理由 |
|--------|------|
| **全タップ可能要素に `:active` 状態を必ず実装** | フィードバックなしはデッドUIに見える |

```css
/* ✅ 必須パターン */
.interactive:active {
  opacity: 0.7;         /* または */
  transform: scale(0.98);  /* または */
  background: rgba(255,255,255,0.05);
}

button:active {
  transform: scale(0.97);
  transition: transform 0.1s ease;
}
```

### 10.7 絵文字の UI 使用禁止（HIG 違反）

| 禁止箇所 | 正しい実装 | 理由 |
|----------|-----------|------|
| **タブバーアイコン** `🏠🤖📊📋` | SVG インラインアイコン | プラットフォーム間でレンダリング差、ダーク/ライトモード非対応 |
| **機能的ヘッダーアイコン** `☀️🔄` | SVG（22×22、stroke currentColor） | 操作可能な要素に絵文字は HIG 違反 |
| **UI状態バッジ** `↔️📈⚡` | テキストラベルまたは SVG | 状態を表すアイコンは正確なベクターを使用 |
| **セクション見出しアイコン** `📊⚖️🎯⚠️🌊` | **省略（テキストのみ）** | アイコンなしの方がスッキリ。許容ではなく禁止 |
| **ランキングメダル** `🥇🥈🥉` | **数字丸バッジ（1/2/3）** | 絵文字はサイズ・色がOS依存。CSS で制御可能な数字バッジが正しい |
| **TP/SL バナーアイコン** `🎯🌸✅` | SVG チェック / クロス | 通知要素のアイコンは SVG で統一 |
| **ログの close reason** `🎯🔴` | `TP` / `SL` テキストバッジ | データ行内アイコンもテキストで明示 |

```css
/* ✅ ランキングバッジの正しい実装 */
.ai-ranking-medal {
  width: 20px; height: 20px;
  border-radius: 50%;
  font-size: 11px; font-weight: 700;
  display: flex; align-items: center; justify-content: center;
}
.ai-ranking-medal--1 { background: rgba(255,204,0,0.18); color: #FFCC00; }
.ai-ranking-medal--2 { background: rgba(174,174,178,0.18); color: #AEAEB2; }
.ai-ranking-medal--3 { background: rgba(188,96,0,0.18); color: #BC6000; }

/* ✅ タブバーの正しい実装 */
.tab-icon svg {
  width: 22px; height: 22px;
  stroke: currentColor; fill: none; stroke-width: 1.8;
}
```

> **例外なし**: UIクロムに絵文字を使う理由はない。ユーザーコンテンツ（チャットメッセージ等）のみ許容。

### 10.8 色のセマンティクス（H6 違反）

| ルール | 理由 |
|--------|------|
| **60%超えの勝率はオレンジ禁止 → 緑を使用** | オレンジ = 警告。65% の勝率をオレンジで表示すると「危険」と誤認させる |
| **オレンジ = 警告・注意のみに使用** | `#FF9F0A` は「まだ不足」「高ボラ」など警告文脈に限定 |
| **勝率しきい値と色のマッピング** | >60%: 緑、50-60%: グレー、<50%: 赤 |

```css
/* ✅ 勝率の色判定ルール */
```

### 10.9 UX 心理学違反

| 法則 | 違反内容 | 修正方針 |
|------|----------|----------|
| **U2 視覚的階層** | 統計ラベルが 9px — 数値より小さく情報構造が壊れる | ラベル 11px、値 14〜16px の明確な階層 |
| **U3 系列位置効果** | 「データ不足警告」が統計タブの中間に埋もれている | 重要な警告は最初か最後に配置 |
| **U1 認知負荷** | 統計専門用語（「Engle-Granger 共和分」等）の説明なし | 用語には必ず 1 行の日本語説明を添える |
| **U9 美的ユーザビリティ** | フォントサイズ・余白の不統一が「安っぽさ」を生む | 一貫したシステムが美観を作る |

### 10.11 flex コンテナ内の二重スペーシング禁止

flex コンテナの `gap` と子要素の `margin-bottom` は **加算される**（flex ではマージン相殺が発生しない）。

| 禁止パターン | 理由 | 修正方針 |
|---|---|---|
| flex `gap: 12px` の子要素に `margin-bottom: 8px` を設定 | 視覚的ギャップが 12+8=20px になる | 子要素の外向きマージンを 0 にし、gap だけで制御 |
| `.card-watchlist` に `padding-top: 12px` + flex gap | カードとリストの間が 12+12=24px + 内部余白で 36px になる | カード側 padding を 0 にし、内部タイトルに `padding: 8px 16px 4px` を付与 |

```css
/* ❌ 禁止 */
.content { gap: 12px; }
.child-element { margin-bottom: 8px; } /* 合計 20px になる */

/* ✅ 正しい */
.content { gap: 12px; }
.child-element { margin-bottom: 0; } /* gap のみで統一 */
```

### 10.10 「ダサい」を防ぐ品質チェック（必須）

UIを実装・改修したら以下を Chrome DevTools MCP でブラウザ確認すること:

```
□ フォントサイズが全て 11px 以上（タブバー以外）
□ テキストカラーが #636366 を使っていない
□ 全タップ要素に :active フィードバックあり
□ padding/margin が 4の倍数（14px・10px・9px がない）
□ 同一階層カードの border-radius が統一されている（16px）
□ UIクロムに絵文字を一切使っていない（ランキングも数字バッジで）
□ オレンジが「良い数値」に使われていない
□ 全インタラクティブ行が min-height: 44px 以上
□ ダークモードでコントラスト比 4.5:1 以上
□ content padding-top が calc(safe-area + 74px) になっているか
□ flex コンテナ内で gap と margin-bottom が二重になっていないか
□ 視覚的に「スッキリ・整然・Apple品質」に見えるか（主観的だが最重要）
```
