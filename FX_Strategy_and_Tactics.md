# FX 戦略・戦術マニュアル
## ― テスタの思想をベースに fx-sim-v1 へ実装する ―

---

## 全体構造

```
戦略（Strategy）= なぜ・何を
  └─ 戦術（Tactics）= どうやって
       └─ 実行ルール（Rules）= 具体的な数値・条件
```

---

## 第1部：戦略（Strategy）

### S1. 生存戦略 ― 退場しない

> **テスタの原則**：「増やそうと思うとリスクが高いことをやってしまう。守りを考えた方が結果として増える」

| 戦略要素 | 定義 | fx-sim-v1 での意味 |
|---|---|---|
| **最大損失限度** | 口座資金に対する1日の最大損失率を固定 | D1テーブルに daily_loss_limit カラムを持ち、到達でその日の取引停止 |
| **1トレード損失上限** | 1回のトレードで失ってよい金額を固定 | ポジションサイズ計算の入力パラメータ |
| **連敗時の縮退** | 連続負けでロットを段階的に縮小 | 3連敗→50%、5連敗→25%、7連敗→当日停止 |
| **ドローダウン管理** | 口座最大からの下落率で全面停止 | 最高水準から10%下落で全ポジション決済＋1週間停止 |

### S2. 期待値戦略 ― 勝率よりも損益比

> **テスタの原則**：「勝ち続けるには100種類以上の手法を相場に合わせて選ぶ」

| 戦略要素 | 定義 | 目標値 |
|---|---|---|
| **リスクリワード比** | 1回の損切り幅に対する利確幅 | 最低 1:1.0（＝勝ちの定義）、推奨 1:2.0 以上、理想 1:3.0 以上 |
| **勝率（RR≥1.0基準）** | 実現RR≥1.0の取引割合 | RRの最大化が唯一の目的。勝率35%でもavgRR≥2.0ならEV正 |
| **期待値の算出** | `(勝率 × 平均利益) - (敗率 × 平均損失)` | 月次で正の期待値を維持 |
| **手法の分散** | 単一手法依存を避ける | 最低3つの手法を環境別に運用 |

### S3. 環境認識戦略 ― 戦う場所を選ぶ

> **テスタの原則**：「なぜ上がるか・下がるかを毎日復習する。外部指標と比較して背景を理解する」

| 環境分類 | 特徴 | 基本方針 |
|---|---|---|
| **強トレンド** | 日足で明確な方向、ADX > 25 | 順張りのみ、逆張り禁止 |
| **弱トレンド** | 方向感はあるが勢い弱い | 押し目/戻りを厳選、ロット控えめ |
| **レンジ** | 明確なサポート/レジスタンスの間で往復 | 上下限での逆張り、中央は見送り |
| **高ボラ（指標・イベント）** | 雇用統計、FOMC、介入警戒 | ノートレード or ロット大幅縮小 |
| **低ボラ（東京午後等）** | 動きが極端に小さい | スキャルピング or 見送り |

## 第2部：戦術（Tactics）

### T1. エントリー戦術

#### T1-1. トレンドフォロー（順張り）

```
条件:
  ① 日足・4H足が同方向（上位足フィルター）
  ② 1H足で押し目/戻りの形成を確認
  ③ 5分足でエントリーシグナル発生
     - 移動平均線のゴールデンクロス/デッドクロス
     - ピンバー（長い下ヒゲ/上ヒゲ）
     - 包み足（アウトサイドバー）

損切り: 直近の安値/高値の少し外側
利確:  損切り幅 × 2 以上の位置、または上位足の抵抗帯
```

#### T1-2. レンジ逆張り

```
条件:
  ① 日足でレンジ形成を確認（ADX < 20）
  ② サポート/レジスタンスに到達
  ③ RSI が70以上 or 30以下で反転シグナル

損切り: レンジの外側（ブレイク確定ライン）
利確:  レンジの反対側 or 中央
ロット: トレンドフォローの 50%
```

#### T1-3. ブレイクアウト

```
条件:
  ① 長期レンジの上限/下限を明確にブレイク
  ② ブレイク時の出来高（ティック数）が増加
  ③ ブレイク後のリテスト（戻り）でエントリー

損切り: ブレイクポイントの内側
利確:  レンジ幅と同等の値幅（N字計算）
注意:  ダマシ（フェイクブレイク）に警戒
```

### T2. ポジション管理戦術

#### T2-1. ポジションサイズ計算

```
ポジションサイズ = 許容損失額 ÷ 損切り幅（pips） ÷ 1pipの価値

例:
  口座 100万円、1トレード許容損失 1%（= 1万円）
  損切り幅 20pips、USD/JPY（1pip = 1,000円/1lot）
  → 1万円 ÷ 20 ÷ 1,000 = 0.5 lot
```

#### T2-2. 分割エントリー・分割決済

```
分割エントリー:
  シグナル発生時に 50% → 追加確認で残り 50%

分割決済:
  第1利確（RR 1:1 到達 = 勝ちの定義）: 50% 決済 → 損切りを建値に移動
  第2利確（RR 1:2 到達 = 推奨目標）: 残り 50% 決済
  ※ 建値ストップにより「負けないトレード」を作る
  ※ RR≥1.0が「勝ち」の最低条件。RR<1.0のプラス決済は「負け」
```

#### T2-3. トレーリングストップ

```
トレンドが伸びた場合:
  - 直近の安値/高値を損切りラインとして追従
  - ATR（Average True Range）× 1.5 を損切り幅として追従
  - 利益が損切り幅と同額に達したら → 建値ストップに移行
```

### T3. 時間帯戦術

> **USD/JPY の特性に最適化**

| 時間帯（JST） | セッション | 特徴 | 戦術 |
|---|---|---|---|
| 8:00-10:00 | 東京オープン | 仲値に向けた円売り傾向 | 短期の円安方向バイアス |
| 10:00-15:00 | 東京午後 | 低ボラティリティ | 基本見送り or 極小ロット |
| 16:00-18:00 | ロンドンオープン | ボラ急増、トレンド発生 | メイン取引時間帯 |
| 21:30-23:00 | NY指標発表 | 指標次第で急変動 | 指標前はノーポジ推奨 |
| 23:00-1:00 | NYアクティブ | 流動性最大 | トレンドフォロー好適 |
| 3:00-7:00 | 早朝 | スプレッド拡大、流動性薄 | 取引禁止 |

### T4. ファンダメンタルズ戦術

#### T4-1. 金利差ドライバー

```
USD/JPY の長期方向性 ≒ 日米金利差の方向

チェックリスト:
  □ FRBの金利見通し（ドットプロット、議事録）
  □ 日銀の政策スタンス（利上げ or 据え置き）
  □ 2年債利回りの方向（短期金利差の代理変数）
  □ 金利先物市場の織り込み度合い
```

#### T4-2. イベントカレンダー管理

```
S級（ノーポジ or 超小ロット）:
  - FOMC 金利決定・声明
  - 米雇用統計
  - 日銀金融政策決定会合
  - CPI（米消費者物価指数）

A級（ロット縮小）:
  - GDP 速報値
  - ISM 製造業/非製造業
  - 小売売上高
  - 為替介入警戒水準接近時

B級（注意しつつ通常運用）:
  - ADP 雇用統計
  - PPI（生産者物価指数）
  - 新規失業保険申請件数
```

### T5. リスク管理戦術

> **テスタの原則**：「最悪な状況になってもいいように備える。経済ショック時はポジションを減らす」

#### T5-1. 損切りルール（絶対遵守）

```
鉄則:
  ① エントリーと同時にストップロス注文を必ず入れる
  ② 一度置いた損切りを遠ざけない（近づけるのはOK）
  ③ 根拠が崩れたら損切りラインに達していなくても手動決済
```

#### T5-2. 日次・週次リスクキャップ

```
日次損失上限:  口座資金の 2%
  → 到達で当日全取引停止

週次損失上限:  口座資金の 5%
  → 到達で当週全取引停止、翌週は 50% ロットで再開

月次損失上限:  口座資金の 10%
  → 到達で当月全取引停止、振り返りと戦略見直し
```

#### T5-3. 相関リスク管理

```
同時保有の制限:
  - 同方向の相関ペアを同時に持たない
    （例: USD/JPY ロング + EUR/JPY ロング = 実質的に円ショート2倍）
  - 最大同時ポジション数: 2
  - 合計エクスポージャーの上限を設定
```

## 第3部：検証ループ（テスタ流 PDCA）

> **テスタの原則**：「株価が下がった理由を分析する。上がると思った理由が間違っていたなら何故か検証する」

### V1. トレード日誌の記録項目

```
[必須項目]
  - 日時、通貨ペア、方向（L/S）
  - エントリー根拠（どの手法、どのシグナル）
  - 環境認識（トレンド/レンジ/高ボラ）
  - ポジションサイズと損切り/利確の位置
  - 結果（pips、金額）
  - 感情状態（冷静/焦り/恐怖/欲）

[検証項目]
  - 根拠は正しかったか？
  - 損切り/利確の位置は適切だったか？
  - 環境認識は正しかったか？
  - ルール違反はなかったか？
```

### V2. 統計分析（D1 に蓄積して自動集計）

```sql
-- 手法別の期待値（RR≥1.0基準）
SELECT strategy,
       COUNT(*) as trades,
       AVG(CASE WHEN realized_rr >= 1.0 THEN 1.0 ELSE 0.0 END) as win_rate,
       AVG(CASE WHEN realized_rr >= 1.0 THEN pnl END) as avg_win,
       AVG(CASE WHEN realized_rr < 1.0 THEN ABS(pnl) END) as avg_loss,
       AVG(realized_rr) as avg_rr,
       SUM(pnl) as total_pnl
FROM trade_logs
WHERE realized_rr IS NOT NULL
GROUP BY strategy;

-- 時間帯別の勝率（RR≥1.0基準）
SELECT session_name,
       COUNT(*) as trades,
       AVG(CASE WHEN realized_rr >= 1.0 THEN 1.0 ELSE 0.0 END) as win_rate,
       AVG(realized_rr) as avg_rr
FROM trade_logs
WHERE realized_rr IS NOT NULL
GROUP BY session_name;

-- 環境別の成績
SELECT market_condition,
       strategy,
       AVG(pnl) as avg_pnl,
       COUNT(*) as sample_size
FROM trade_logs
GROUP BY market_condition, strategy
HAVING sample_size >= 20;
```

### V3. 戦略改善サイクル

```
週次レビュー:
  - 今週の成績サマリー
  - ルール違反の有無
  - 各手法の期待値変化

月次レビュー:
  - 手法別・時間帯別・環境別の統計更新
  - 期待値がマイナスの手法 → 改善 or 停止
  - 新しい手法の仮説 → 過去データで検証 → 小ロットで試行

四半期レビュー:
  - 戦略レベルの見直し
  - リスクパラメータの再調整
  - Gemini API シグナルの精度分析
```

## 第4部：fx-sim-v1 実装マッピング

### アーキテクチャとの対応

| 戦略・戦術 | fx-sim-v1 コンポーネント | 実装方法 |
|---|---|---|
| S1. 生存戦略 | Worker のリスク管理ロジック | 日次/週次/月次の損失チェックを取引前に実行 |
| S2. 期待値戦略 | D1 の集計クエリ | trade_logs テーブルから自動算出 |
| S3. 環境認識 | Gemini API への入力 | プロンプトに環境分類ロジックを組み込む |
| T1. エントリー | Gemini API のシグナル生成 | 複数手法のシグナルを生成し、環境に合った手法を選択 |
| T2. ポジション管理 | Worker の注文ロジック | サイズ計算、分割決済の自動化 |
| T3. 時間帯 | Cron Trigger の制御 | 時間帯により取引可否とロット倍率を変動 |
| T4. ファンダ | Gemini API + 外部データ | 経済指標カレンダーAPIを参照 |
| T5. リスク管理 | Worker のガードロジック | 全エントリー前にリスクチェックを通過させる |
| V1-V3. 検証 | D1 + 定期バッチ | 日次/週次サマリーを自動生成 |

### D1 スキーマ（追加提案）

```sql
-- 環境認識ログ
CREATE TABLE market_conditions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  timestamp TEXT NOT NULL,
  pair TEXT DEFAULT 'USD/JPY',
  condition TEXT NOT NULL,       -- trend_strong/trend_weak/range/high_vol/low_vol
  adx_value REAL,
  atr_value REAL,
  session TEXT,                  -- tokyo/london/newyork
  notes TEXT
);

-- リスク管理ステート
CREATE TABLE risk_state (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  date TEXT NOT NULL UNIQUE,
  daily_pnl REAL DEFAULT 0,
  daily_trades INTEGER DEFAULT 0,
  consecutive_losses INTEGER DEFAULT 0,
  lot_multiplier REAL DEFAULT 1.0,
  is_locked INTEGER DEFAULT 0,   -- 1 = 取引停止
  lock_reason TEXT
);
```

## 付録：テスタ語録とFXへの適用

| テスタの言葉 | FXでの解釈 |
|---|---|
| 「守りを考えた方が結果として増える」 | 損切り厳守・ロット管理が最優先 |
| 「100種類以上の手法を持っている」 | 環境ごとに複数の手法を使い分ける |
| 「危ないと思ったら反射的にロスカット」 | 逆指値は必ず設定、根拠崩壊で即撤退 |
| 「結果的に戻っても、最悪に備える」 | 含み損のナンピン禁止、希望的観測排除 |
| 「毎日復習する。なぜ動いたかを検証する」 | トレード日誌と統計分析の継続 |
| 「真剣に長く取り組めば上達する」 | ペーパートレードで経験を積み、統計を蓄積する |
| 「いきなり全財産を投じない」 | 小ロットから開始、実績に応じて段階的に拡大 |
