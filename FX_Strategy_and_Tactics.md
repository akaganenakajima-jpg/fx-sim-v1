## 第2部：戦略（Strategy）― テスタ哲学のFX実装

> 以下はテスタの哲学をFXトレードに応用するための戦略。
> 具体的な数値・指標はFXトレード理論（一般的な文献）に基づく。

### S1. 生存戦略 ― 退場しない（P1, P3, P6 の実装）

| 戦略要素 | 定義 | fx-sim-v1 での意味 | 根拠 |
|---|---|---|---|
| **最大損失限度** | 口座資金に対する1日の最大損失率を固定 | D1テーブルに daily_loss_limit カラムを持ち、到達でその日の取引停止 | Elder (1993) 「2%ルール」 |
| **1トレード損失上限** | 1回のトレードで失ってよい金額を固定 | ポジションサイズ計算の入力パラメータ | Kelly (1956) の保守的適用 |
| **連敗時の縮退** | 連続負けでロットを段階的に縮小 | 連敗が続いたらロット縮小→手法と環境の適合性を検証 | Elder (1993), Van Tharp (1998) |
| **ドローダウン管理** | 口座最大からの下落率で停止判断 | DD率に応じた段階的対応 | テスタ: 「ピークから4割減で投資をやめる」 |

### S2. 期待値戦略 ― 勝率よりも損益比（P4 の実装）

| 戦略要素 | 定義 | 目標値 |
|---|---|---|
| **リスクリワード比** | 1回の損切り幅に対する利確幅 | 最低 1:1.0（＝勝ちの定義）、推奨 1:2.0 以上、理想 1:3.0 以上 |
| **勝率（RR>=1.0基準）** | 実現RR>=1.0の取引割合 | RRの最大化が唯一の目的。勝率35%でもavgRR>=2.0ならEV正 |
| **期待値の算出** | `(勝率 x 平均利益) - (敗率 x 平均損失)` | 月次で正の期待値を維持 |
| **手法の分散** | 単一手法依存を避ける（P4） | 最低3つの手法を環境別に運用 |

### S3. 環境認識戦略 ― 戦う場所を選ぶ（P4, P5 の実装）

| 環境分類 | 特徴 | 基本方針 |
|---|---|---|
| **強トレンド** | 日足で明確な方向 | 順張りのみ、逆張り禁止 |
| **弱トレンド** | 方向感はあるが勢い弱い | 押し目/戻りを厳選、ロット控えめ |
| **レンジ** | 明確なサポート/レジスタンスの間で往復 | 上下限での逆張り、中央は見送り |
| **高ボラ（指標・イベント）** | 雇用統計、FOMC、介入警戒 | ノートレード or ロット大幅縮小 |
| **低ボラ（東京午後等）** | 動きが極端に小さい | スキャルピング or 見送り |

## 第3部：戦術（Tactics）― FXトレード理論

> 以下はFXトレード理論に基づく一般的な戦術。
> テスタ独自の手法ではなく、テスタの哲学（特に P2, P3, P4）に沿う形で選定した。

### T1. エントリー戦術

#### T1-1. トレンドフォロー（順張り）

```
条件:
  1. 日足・4H足が同方向（上位足フィルター）
  2. 1H足で押し目/戻りの形成を確認
  3. 5分足でエントリーシグナル発生
     - 移動平均線のゴールデンクロス/デッドクロス
     - ピンバー（長い下ヒゲ/上ヒゲ）
     - 包み足（アウトサイドバー）

損切り: 直近の安値/高値の少し外側
利確:  損切り幅 x 2 以上の位置、または上位足の抵抗帯
```

#### T1-2. レンジ逆張り

```
条件:
  1. 日足でレンジ形成を確認
  2. サポート/レジスタンスに到達
  3. 反転シグナル（プライスアクション）

損切り: レンジの外側（ブレイク確定ライン）
利確:  レンジの反対側 or 中央
ロット: トレンドフォローの 50%
```

#### T1-3. ブレイクアウト

```
条件:
  1. 長期レンジの上限/下限を明確にブレイク
  2. ブレイク時の出来高（ティック数）が増加
  3. ブレイク後のリテスト（戻り）でエントリー

損切り: ブレイクポイントの内側
利確:  レンジ幅と同等の値幅（N字計算）
注意:  ダマシ（フェイクブレイク）に警戒
```

### T2. ポジション管理戦術

#### T2-1. ポジションサイズ計算

```
ポジションサイズ = 許容損失額 / 損切り幅（pips） / 1pipの価値

例:
  口座 100万円、1トレード許容損失 1%（= 1万円）
  損切り幅 20pips、USD/JPY（1pip = 1,000円/1lot）
  -> 1万円 / 20 / 1,000 = 0.5 lot
```

#### T2-2. 分割エントリー・分割決済

```
分割エントリー:
  シグナル発生時に 50% -> 追加確認で残り 50%

分割決済:
  第1利確（RR 1:1 到達 = 勝ちの定義）: 50% 決済 -> 損切りを建値に移動
  第2利確（RR 1:2 到達 = 推奨目標）: 残り 50% 決済
  ※ 建値ストップにより「負けないトレード」を作る
  ※ RR>=1.0が「勝ち」の最低条件。RR<1.0のプラス決済は「負け」
```

#### T2-3. トレーリングストップ

```
トレンドが伸びた場合:
  - 直近の安値/高値を損切りラインとして追従
  - ATR（Average True Range） x 1.5 を損切り幅として追従
  - 利益が損切り幅と同額に達したら -> 建値ストップに移行
```

### T5. リスク管理戦術（P2, P3, P8 の実装）

#### T5-1. 損切りルール（テスタ哲学の核心 → 絶対遵守）

```

  1. エントリーと同時にストップロス注文を必ず入れる
```

#### T5-2. 日次・週次リスクキャップ

> **注記**: 以下の具体的数値はテスタの発言ではなく、Alexander Elder『投資苑』(1993)等の

```

  -> 到達で当日全取引停止

週次損失上限:  口座資金の 5%
  -> 到達で当週全取引停止、翌週は 50% ロットで再開

月次損失上限:  口座資金の 10%
  -> 到達で当月全取引停止、振り返りと戦略見直し
```

```

  - 同方向の相関ペアを同時に持たない

  - 最大同時ポジション数: 2
  - 合計エクスポージャーの上限を設定
```

### V2. 統計分析（D1 に蓄積して自動集計）

```sql
-- 手法別の期待値（RR>=1.0基準）
SELECT strategy,
       COUNT(*) as trades,
       AVG(CASE WHEN realized_rr >= 1.0 THEN 1.0 ELSE 0.0 END) as win_rate,
       AVG(CASE WHEN realized_rr >= 1.0 THEN pnl END) as avg_win,
       AVG(CASE WHEN realized_rr < 1.0 THEN ABS(pnl) END) as avg_loss,
       AVG(realized_rr) as avg_rr,
       SUM(pnl) as total_pnl
FROM trade_logs
WHERE realized_rr IS NOT NULL
GROUP BY strategy;

-- 時間帯別の勝率（RR>=1.0基準）
SELECT session_name,
       COUNT(*) as trades,
       AVG(CASE WHEN realized_rr >= 1.0 THEN 1.0 ELSE 0.0 END) as win_rate,
       AVG(realized_rr) as avg_rr
FROM trade_logs
WHERE realized_rr IS NOT NULL
GROUP BY session_name;

-- 環境別の成績
SELECT market_condition,
       strategy,
       AVG(pnl) as avg_pnl,
       COUNT(*) as sample_size
FROM trade_logs
GROUP BY market_condition, strategy
HAVING sample_size >= 20;
```

## 第5部：fx-sim-v1 実装マッピング

### アーキテクチャとの対応

| テスタ哲学・戦略・戦術 | fx-sim-v1 コンポーネント | 実装方法 |
|---|---|---|
| P1-P3 生存哲学 → S1 | Worker のリスク管理ロジック | 日次/週次/月次の損失チェックを取引前に実行 |
| P4 手法分散 → S2 | D1 の集計クエリ | trade_logs テーブルから自動算出 |
| P5 雰囲気トレード → S3 | Gemini API への入力 | プロンプトに環境分類ロジックを組み込む |
| T1 エントリー | Gemini API のシグナル生成 | 複数手法のシグナルを生成し、環境に合った手法を選択 |
| T2 ポジション管理 | Worker の注文ロジック | サイズ計算、分割決済の自動化 |
| T3 時間帯 | Cron Trigger の制御 | 時間帯により取引可否とロット倍率を変動 |
| T4 ファンダ | Gemini API + 外部データ | 経済指標カレンダーAPIを参照 |
| P2 損切り哲学 → T5 | Worker のガードロジック | 全エントリー前にリスクチェックを通過させる |
| P9 記録と復習 → V1-V3 | D1 + 定期バッチ | 日次/週次サマリーを自動生成 |

```sql
CREATE TABLE risk_state (
  id INTEGER PRIMARY KEY AUTOINCREMENT,

  daily_trades INTEGER DEFAULT 0,
  consecutive_losses INTEGER DEFAULT 0,
  lot_multiplier REAL DEFAULT 1.0,
  is_locked INTEGER DEFAULT 0,   -- 1 = 取引停止

);
```
