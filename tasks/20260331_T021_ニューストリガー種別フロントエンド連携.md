# T021: ニューストリガー種別（EMERGENCY/TREND）のフロントエンド連携

## 背景・目的
- 「イランのアルミニウム…」ニュースが EMERGENCY（news_trigger_log.trigger_type）なのに、フロントエンドでは「トレンド · score 7.5」と誤表示
- 原因: フロントエンドが `composite_score`（7軸加重平均 = 7.5）しか参照できず、`trigger_type` を知る手段がなかった

## スコアの食い違いの本質
- `composite_score`（7.5）: 7軸加重平均 → 鮮度・重複排除で薄まった総合点
- `relevance`（9）× `sentiment`（8）: AI個別軸 → EMERGENCY判定はこちらで行う
- **Stage 1は総合点でフィルタ（処理すべきか）、Stage 2は個別点で判定（取引を変えるか）** → 設計は正しい
- 問題は trigger_type がフロントエンドに渡っていなかっただけ

## 修正内容

### 1. API（src/api.ts）
- `StatusResponse` に `newsTriggers` フィールドを追加 ← **実装済み**
- レスポンスオブジェクトに `newsTriggers` を含める ← **実装済み**

### 2. フロントエンド（src/app.js.ts）
- `renderNewsTab()` で `data.newsTriggers` からタイトル→trigger_type のルックアップマップを構築
- `acceptedAsAnalysis` マッピング時に `triggerType` フィールドを付与
- `newsCard()` でバッジ判定を変更:
  - `n.triggerType === 'EMERGENCY'` → 緊急バッジ
  - `n.triggerType === 'TREND_INFLUENCE'` → トレンドバッジ
  - trigger_type なし → スコアベースのフォールバック
- KPI の緊急件数カウントも trigger_type ベースに変更

### 3. 時系列ソート修正
- ニュースタブの3セクション全てが新しい順に並ぶことを確認・修正

## 注意事項
- `news_trigger_log` は直近12時間のみ取得（既存クエリ）→ それ以前の記事はフォールバック判定
