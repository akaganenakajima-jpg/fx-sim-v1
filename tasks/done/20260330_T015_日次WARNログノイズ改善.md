# T015 日次エントリー件数不足 WARN ログのノイズ改善

**作成日**: 2026-03-30
**優先度**: 🟡 低
**発見経緯**: E2E Cycle 8〜10（バックエンドE2Eチェック）

---

## 背景・目的

「日次BUY+SELL件数不足: XX件（目標>=100）」WARN が毎 cron（1分ごと）記録されている。
30分で 100件以上の WARN が蓄積し、system_logs が汚染されて本物の異常が埋もれるリスクがある。

## 現象

```
直近30分のWARNログ種別（代表）:
  日次BUY+SELL件数不足: 24件 (目標>=100) → 79件
  日次BUY+SELL件数不足: 15件 (目標>=100) → 14件
  日次BUY+SELL件数不足: 10件 (目標>=100) → 10件
  日次BUY+SELL件数不足: 0件 (目標>=100)  → 10件
合計: 30分で 115件のWARN
```

一日の序盤（UTC 03:00 = JST 12:00）では目標100件は不可能なため、
毎 cron で WARN が出るのは設計上の問題。

## 修正方針

時刻帯に応じた段階的目標値に変更する（時刻比例チェック）:

```typescript
// 現在の UTC 時刻から当日の経過割合でチェック
const elapsed = (utcHour * 60 + utcMin) / (24 * 60);  // 0.0〜1.0
const dynamicTarget = Math.floor(DAILY_TARGET * elapsed * 1.1); // 少し余裕をもたせる
if (todayCount < dynamicTarget) {
  WARN(`日次BUY+SELL件数不足: ${todayCount}件 (現時点目標: ${dynamicTarget}件)`);
}
```

あるいは WARN 発火頻度を下げる（1時間に1回など）。

## 対象ファイル

`src/index.ts` の MONITOR WARN 発火箇所
```bash
grep -n "日次BUY+SELL件数不足" src/index.ts
```

## 完了条件

- 日中の通常稼働時に「件数不足 WARN」が 1時間あたり 5件以下に収まること
- EOD（UTC 20:00以降）に目標未達の場合のみ WARN が出ること
