# T014 SPRT（逐次確率比検定）6日間停止の調査・修正

**作成日**: 2026-03-30
**優先度**: 🟠 中
**発見経緯**: E2E Cycle 8〜10（バックエンドE2Eチェック）

---

## 背景・目的

`evaluateRecovery()`（SPRT: Sequential Probability Ratio Test）が 2026-03-24 以降
約6日間呼ばれていない。DD回復時のロット段階昇格判定が機能していない状態。

## 現象

```sql
-- risk_state テーブルの実測値
sprt_last_evaluated_id : 503   updated_at: 2026-03-24T05:52:34Z
sprt_log_likelihood    : 0     updated_at: 2026-03-24T05:52:35Z
```

- positions テーブルの最新 CLOSED id は 608（約100件分の評価が積み残し）
- sprt_last_evaluated_id=503 から 608 まで 105件が未評価

## 推定原因

1. `evaluateRecovery()` が `src/index.ts` の cron フローで条件分岐によりスキップされている
2. SPRT が DD レベル != NORMAL の場合のみ呼ばれる設計になっており、
   現在 DD レベルが NORMAL（DD < 5%）のためスキップされている可能性

## 調査手順

1. `src/index.ts` で `evaluateRecovery` の呼び出し箇所を確認
   ```bash
   grep -n "evaluateRecovery" src/index.ts src/risk-manager.ts
   ```
2. 呼び出し条件（if 文）を確認し、現在の DD レベルでスキップされるか判定
3. `src/risk-manager.ts` の `evaluateRecovery()` の動作仕様を再確認

## 修正方針

- SPRT は DD レベルに関わらず常時評価すべきか、DD 発生時のみか、仕様を確認してから修正
- 常時評価が正しい場合: 呼び出し条件の if 文を除去または緩和
- DD 発生時のみ評価が正しい場合: 現状は設計通りでドキュメントに明記

## 完了条件

- `sprt_last_evaluated_id` が直近のトレードに追従して更新されること
  または設計上スキップが正しいことが確認・文書化されること
