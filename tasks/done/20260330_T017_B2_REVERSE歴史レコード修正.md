# T017 B2_REVERSE 歴史的レコード（id=606）の pnl 誇大計上修正

**作成日**: 2026-03-30
**優先度**: 🟡 低（仮想取引のため実害なし）
**発見経緯**: E2E Cycle 8 Round 1（Agent B）

---

## 背景・目的

今セッション（2026-03-30）で B2_REVERSE クローズ時の pnl 計算バグを修正したが、
修正前に記録された id=606（Gold B2_REVERSE）が誤った値のまま DB に残っている。
累積 PnL 統計が約 ¥2,169 誇大計上されている。

## 誤レコードの内容

```
id: 606
pair: Gold
direction: BUY
entry_rate: 4466.2
close_rate: 4490.3
pnl: 2410.0  ← 誤り（multiplier=100 が適用された）
log_return: NULL
realized_rr: NULL
close_reason: B2_REVERSE
```

正しい値:
```
pnl = (4490.3 - 4466.2) × pnlMultiplier(10) = 241.0
log_return = log(4490.3 / 4466.2) = 0.00539...
realized_rr = calcRealizedRR('BUY', 4466.2, 4490.3, sl_rate) の結果
```

（sl_rate が NULL のため realized_rr は計算不可 → NULL のまま許容）

## 修正 SQL

```sql
UPDATE positions
SET
  pnl = (4490.3 - 4466.2) * 10,
  log_return = 0.005390
WHERE id = 606;
```

実行コマンド:
```bash
wrangler d1 execute fx-sim-v1-db --remote --command \
  "UPDATE positions SET pnl = 241.0, log_return = 0.005390 WHERE id = 606"
```

## 影響範囲

- 修正後の累積 PnL: `5387.63 - 2410 + 241 = 3218.63` → 約 ¥2,169 減少
- HWM（15,614円）はリアルタイム残高に基づくため、実際の影響は軽微
- 勝率・realized_rr 統計には影響なし（NULL のまま）

## 完了条件

- `SELECT pnl FROM positions WHERE id=606` が 241.0 を返すこと
