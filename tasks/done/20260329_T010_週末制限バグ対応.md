# 20260329_T010: 週末クローズ中のFX銘柄誤エントリーバグ対応

## 背景・目的
週末（土日）に市場が閉じているにもかかわらず、フロントエンドに FX/株指数のOPENポジションが表示されていた。
調査の結果、Path B（AI判断）の銘柄フィルタが週末クローズ状態を考慮していないことが判明。

---

## 根本原因

**d93717e（Path A廃止コミット, 2026-03-25頃）** で `runAIDecisions`（旧 Path A）が削除された際、
その関数が持っていた `cryptoOnlyMode` による FX 除外ガードが消え、後継の `runPathB` に引き継がれなかった。

| 削除された関数 | 消えたガード | 影響 |
|---|---|---|
| `runAIDecisions`（Path A） | `if (cryptoOnlyMode && !CRYPTO_PAIRS.has(i.pair)) continue` | 週末にFX銘柄が誤エントリー |

**検出が遅れた理由**: Path A 廃止は平日に作業・テストされたため、週末クローズ状態での動作確認機会がなかった。

---

## 対応内容

### 応急処置（PR #117）
- `runPathB` 内で `getTradeableInstruments()` を呼び出し、週末クローズ中は BTC/ETH/SOL のみ対象にフィルタ

### IPA基準 5層 再発防止策（同PR）

| Layer | 内容 |
|---|---|
| 1. 要件 | CLAUDE.md に「週末市場クローズ制約」セクションを新設 |
| 2. 設計 | `CRYPTO_PAIRS` を `weekend.ts` に集約（Single Source of Truth）、`getTradeableInstruments()` を新設 |
| 3. 実装 | `runPathB` に JSDoc で週末制約の警告を明記 |
| 4. テスト | `weekend.test.ts` 新規作成（12件: 境界値・フィルタ・再発防止） |
| 5. 変更管理 | CLAUDE.md に大規模改修チェックリスト + 過去の教訓テーブルを追加 |
