# fx-sim プロジェクト「勝率・RR・win」用語網羅探索報告

**実施日**: 2026-03-25  
**対象範囲**: src/*.ts, docs/*.md, FX_Strategy_and_Tactics.md  
**新定義**: RR≥1.0=勝ち（既存： RR≥1.5=正常, 1.0≤RR<1.5=削減, RR<1.0=却下）

## 10. 結論

新定義「RR≥1.0=勝ち」は現在のシステム設計（RR≥1.5=正常）と深い矛盾を持つ。**最も影響を受けるのは：**

1. **sanity.ts のRR検証ロジック** — 段階廃止で単純化可能
2. **param-review.ts と FX_Strategy_and_Tactics.md** — 目標RR引き下げが必須
3. **position.ts の Kelly実装** — ポジションサイズに直結する大きな影響

**実装には3段階（定義確定→コア実装→運用検証）を推奨。**
