## 🔴 P1: 即効性 × 高インパクト（次に着手すべき）

### T004-01: D1インデックス追加 [IPA/DB]
- **理由**: データ増加でクエリ劣化する前に予防。実装コスト最小
- **作業**: 4つのインデックスを追加
  ```sql
  CREATE INDEX idx_positions_status ON positions(status);
  CREATE INDEX idx_positions_pair_status ON positions(pair, status);
  CREATE INDEX idx_decisions_pair_created ON decisions(pair, created_at DESC);
  CREATE INDEX idx_system_logs_id_desc ON system_logs(id DESC);
  ```
- **工数**: 15分
- **効果**: 全クエリの応答時間改善、API応答速度向上

### T004-02: ROIブートストラップ 95% CI [統計]
- **理由**: 勝率CIは実装済みだがROIのCIが未実装。投資判断の核心指標
- **作業**: stats.ts に `bootstrapROI()` を追加、api.ts で statistics に含める、ダッシュボード表示
- **工数**: 1時間
- **効果**: 「ROI +X% だがCI [-Y%, +Z%]」で信頼性が一目で分かる

### T004-03: AIモデル評価指標（Brier Score + 的中率トラッキング）[統計]
- **理由**: 「AIが当たっているか」の定量評価がゼロ。最大の盲点
- **作業**:
  - decisions テーブルに `outcome` カラム追加（実際の結果を記録）
  - ポジションクローズ時に対応する decision の outcome を更新
  - Brier Score・方向的中率を stats.ts に追加
  - ダッシュボード統計タブに表示
- **工数**: 3時間
- **効果**: AIの判断精度を統計的に評価可能に

### T004-04: TP/SLレートのサニティチェック [IPA/安全性]
- **理由**: AIが極端な値を返すと大損失リスク。実弾移行前に必須
- **作業**: gemini.ts でTP/SL距離の上限・下限を検証（例: 0.1%〜5%以内）
- **工数**: 30分
- **効果**: 異常値による意図しない大損失を防止

## 🟡 P2: 中期改善（P1完了後）

### T004-05: ベースライン比較（ランダム戦略バックテスト）[統計]
- **理由**: AI戦略がランダムより優れている証明がない
- **作業**:
  - 既存 decisions データからランダム戦略をシミュレート
  - Mann-Whitney U検定で有意差を判定
  - 統計タブに比較表示
- **工数**: 2時間
- **効果**: 「AIはコイントスより優れているか？」にp値で回答

### T004-06: 銘柄間相関モニタリング [統計]
- **理由**: 17銘柄の同時ポジションでリスク集中の可能性
- **作業**:
  - 銘柄別リターン相関行列を算出（ピアソン相関）
  - 相関 > 0.7 の銘柄ペア同時保有時に警告
  - position.ts のオープン判定に組み込み
- **工数**: 3時間
- **効果**: ポートフォリオリスク分散度の可視化

### T004-07: 多重比較補正（Bonferroni/FDR）[統計]
- **理由**: 17銘柄の勝率を個別検定すると偽陽性が増える
- **作業**: stats.ts に `adjustedPValues()` を追加、銘柄スコアに反映
- **工数**: 1時間
- **効果**: 「たまたま勝っている銘柄」の識別

### T004-08: マイグレーション管理の体系化 [IPA/運用]
- **理由**: `schema_v2_migrated` フラグ方式は拡張性が低い
- **作業**: `schema_version` テーブルを導入、マイグレーション関数を体系化
- **工数**: 2時間
- **効果**: 今後のスキーマ変更を安全に管理

### T004-09: 対数リターン（log-return）への移行 [統計]
- **理由**: 金融時系列のI(1)過程には対数差分が理論的に正しい
- **作業**: PnL計算を `log(close/entry)` ベースに変更
- **工数**: 2時間
- **注意**: 既存データとの整合性維持が必要

### T004-10: 残差分析（SL損切りパターン分析）[統計]
- **理由**: 特定条件下（高VIX・特定時間帯）でのAI精度劣化を検出
- **作業**: SL損切り時の特徴量を集計・パターン抽出
- **工数**: 2時間
- **効果**: 「この条件ではAIが弱い」を可視化
