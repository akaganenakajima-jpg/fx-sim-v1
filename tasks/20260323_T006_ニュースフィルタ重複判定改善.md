# T006: ニュースフィルタ（Haiku）重複判定の改善

## 背景・目的

news_raw の100件分析で、**採用1件 / 不採用99件**（採用率1%）という極端な結果を確認。
不採用理由の95%が「重複」だが、実際には**同テーマの異なる角度・新事実を持つ別記事**が多く含まれている。

## 問題の本質

Haikuフィルタが「同テーマ = 重複」と判定しているが、トレード判断には**同テーマでも異なるファクト**が重要。

### 例: イラン関連で全て「重複」扱いされた記事群
- id:17「ベッセント財務長官ホルムズ攻撃継続」→ **政府方針の一次情報**
- id:1414「ホルムズ海峡2週間の期限」→ **経済影響の具体的タイムライン**
- id:7533「投資家は波乱に備える」→ **市場センチメントの変化**
- id:6217「トルコ外相、仲介意欲」→ **外交進展の新事実**

## 改善案

### Haikuフィルタプロンプト修正
1. 「同テーマ=重複」→「**同ファクト=重複**」に判定基準を変更
2. 同テーマでも**新しい事実・数値・発言者**があれば採用する
3. 重複判定は「タイトルが実質同一」の場合のみ適用
4. 目標採用率: 1% → **10-20%**（1回のフィルタで2-5件程度）

### 対象ファイル
- `src/news.ts`: `filterAndTranslateWithHaiku()` のプロンプト

## D1 news_raw 保存量

- **上限制御**: 件数上限なし。TTLパージ（7日）のみ
- **現在**: 107件（約5時間分）
- **推定最大量**: 7日 × 24時間 × 約20件/時 = **約3,300件**
- **hash UNIQUE制約**: 重複タイトルは自動排除（INSERT OR IGNORE）
