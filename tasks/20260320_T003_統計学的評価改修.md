# T003: 統計学的評価に基づく改修タスク

## 背景
統計学知識ベース（時系列分析・確率分布・統計的推定・回帰/GLM・確率過程・ベイズ統計・実験計画法）によるシステム総合評価（スコア69.1/100）から抽出した改善項目。

## 改修項目

### 🔴 優先度: 高（統計的信頼性の根幹）

- [ ] **パフォーマンス統計に信頼区間を追加**
  - 勝率: Wilson区間 95% CI を算出・ダッシュボード表示
  - Sharpe比: 標準誤差 `√((1 + S²/2) / n)` で有意性を表示
  - ROI: ブートストラップ 95% CI
  - 目的: 「統計的に優位かまだ分からない」が一目で分かるようにする
  - 現状: 125件で勝率58.4% → 95% CI = [49.3%, 67.0%]（50%を含む）

- [ ] **ベースライン比較（ランダム戦略バックテスト）**
  - 同期間の市場データで「コイントスBUY/SELL（p=0.5）」をシミュレート
  - AI戦略との PnL分布差を Mann-Whitney U検定で検定
  - 結果をダッシュボード統計タブに表示

- [ ] **テールリスク定量化（VaR/CVaR）**
  - PnL分布の下位5%タイルから「1日で最大いくら失いうるか」を算出
  - 現在のキルスイッチ（¥500/日）が適切かをデータで裏付ける
  - ダッシュボードに VaR(95%) / CVaR(95%) を表示

- [ ] **AIモデル評価指標の導入**
  - Brier Score（確率的予測の精度）
  - BUY/SELL分類のAUC-ROC相当指標
  - 「AIの判定が当たったか」の系統的トラッキング

### 🟡 優先度: 中

- [ ] **ケリー基準によるポジションサイジング完成**
  - 現状: 勝率のみでlot決定（0.5x〜2.0x）
  - 改善: `f* = winRate - (1-winRate)/avgRR` でRR比も反映
  - 銘柄別に最適ベットサイズを算出

- [ ] **マルコフ遷移行列の推定**
  - WIN/LOSE状態の遷移確率を推定
  - 連敗確率・定常分布をダッシュボードに表示
  - DRAWDOWN検知の理論的裏付け

- [ ] **銘柄間相関モニタリング**
  - 17銘柄のリターン相関行列を日次計算
  - ポートフォリオ全体のリスク分散度を可視化
  - 相関の高い銘柄ペア（EUR/USD & GBP/USD等）の同時ポジションを警告

- [ ] **多重比較補正**
  - 17銘柄の勝率を同時検定する際のBonferroni/FDR補正
  - 「たまたま勝っている銘柄」の識別

- [ ] **対数リターン（log-return）への移行**
  - 現状: レート変化を単純差分で判定
  - 改善: 金融時系列のI(1)過程に対応した対数差分を使用

- [ ] **トンプソン・サンプリングの導入**
  - 銘柄選択にBeta分布サンプリングを導入
  - 探索（低確信銘柄）vs 活用（高勝率銘柄）のバランス

- [ ] **残差分析（SL損切りパターン分析）**
  - SL損切り時の特徴量パターン（VIX・時間帯・トレンド方向）を集計
  - 特定条件下でのAI判定精度劣化を検出

### 🟢 優先度: 低

- [ ] **GARCH(1,1)ボラティリティ推定**
  - VIX unavailable時でもSL幅を動的調整可能にする
  - 自前リターン系列からボラティリティクラスタリングを検出

- [ ] **共和分関係の検証**
  - EUR/USD と GBP/USD、Gold と US10Y 等のペア間長期均衡
  - VECM（ベクトル誤差修正モデル）でペアトレード戦略の検討

- [ ] **状態空間モデル（カルマンフィルタ）**
  - 市場レジーム（トレンド/レンジ/高ボラ）をリアルタイム推定
  - レジームに応じた動的戦略切り替え

- [ ] **階層ベイズモデル**
  - 為替ペア群・株価指数群などカテゴリ間の情報共有
  - 新銘柄追加時のコールドスタート問題を緩和

- [ ] **実験の再現性確保**
  - AIプロンプトのバージョニング
  - 判定の非決定性を記録（temperature、使用プロバイダー）

- [ ] **検出力分析に基づくサンプルサイズ設計**
  - 勝率50%→55%の差を検出するには n≈780件必要（α=0.05, power=0.8）
  - 現在125件では粗い推定。目標取引数をロードマップに反映
